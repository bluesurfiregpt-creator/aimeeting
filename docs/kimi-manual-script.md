# § 2. 系统 背景 (你 必须 充分 理解)

## 2.1 项目 名称

**Aimeeting · AI 专家天团**

(若 客户 行业 偏好 不同, 可适当 调整 副标; 主名 Aimeeting 不动)

## 2.2 设计 初衷 — 我们 为 什么 做 这个

**痛点 三连** (大量 政府 / 国企 / 企业 中层 的 真实 抱怨):

| 痛点 | 老 状态 | 后果 |
|------|--------|------|
| **议程 跑偏** | 议程 只是 会前 一张纸, 开会 时 没人 看, 容易 闲扯 | 计划 1 小时 的会, 跑成 2 小时, 后面 议题 草草 收尾 |
| **决策 黑箱** | 几个 方案 反复 讨论 没人 拍板, 散会 后 大家 各自 解读 | 下次 又得 重 讨论, 跨 部门 推进 卡死 |
| **沉淀 流失** | 重要 决策 / 经验 仅 留在 个人 大脑里, 半年 后 找不到 谁 说 了 啥 | 同样 的 错误 反复 犯, 老员工 离职 一片 真空 |

传统 会议 软件 (录音 转文字 + 纪要 模板) 解决 不了 这些 — 它们 是 "记录 工具", 我们 要 的 是 **"AI 专家 真 参与 + 经验 累积 越用 越 聪明"**.

## 2.3 核心 心智 模型 (你 一开口 就 应 让 客户 装进 脑子)

```
🤖 AI 专家  =  数字员工 (有 性格 / 有 专业 / 有 记忆)
 ┣ 📚 书架  (KB)      — 查 得 到 的 资料 (规章 / 文档 / 流程)
 ┣ 🧠 经验  (Memory)  — 已 记住 的 事 (上次 议过 / 决策 过 / 教训)
 ┗ 🎙️ 会议            — 触发 AI 工作 + 产出 上面 两者
```

这 不是 比喻 — 是 系统 真 实现 的 数据 模型. 每个 AI 专家 真 有 自己 的 书架 (KB) 和 经验 (Memory). 越 开 会, 书架 越 厚, 经验 越 丰富, AI 越 像 一个 老员工.

## 2.4 三 大 价值 主张 (你 必须 在 手册 开头 就 立住)

1. **不 是 录音转文字 — 是 AI 专家 真 跟 你 一起 开会**
   - 政策 法规 专家 / 数据 分析 专家 / 项目 管理 专家 同时 在 会场
   - 你 谈 到 数据 出境 → 政策 专家 自动 开口 提合规 风险
   - 召之 即来, 不需要 真请 一个 法律 顾问 坐 一下午

2. **越 开 越 聪明 — 上次 议过 的 事, 这次 自动 引用**
   - 6 个月 前 决策 过 "A 区 物业费 涨 5%", 今天 又 议 类似 → AI 主动 把 上次 决策 + 后续 反馈 提出来
   - 半年 后 老员工 离职, AI 仍 记得 部门 所有 决策 + 教训

3. **全程 可 追溯 — 每条 建议 / 经验 都 能 链回 原话**
   - AI 抽 出来 的 待办 → 一键 跳到 哪 几句 话 抽 出来 的
   - 入库 的 经验 → 一键 跳到 半年 前 哪 场 会 议 的 哪 几 句 原话
   - 不 信 AI 总结 时, 一键 验证

# § 3. 系统 整体 架构 (高层 概念, 给 客户 看 的 那 种)

> 客户 不 关心 技术 细节. 但 你 应 让 客户 知道: 系统 不 是 一个 单点 工具, 是 一套 完整 闭环.

## 3.1 闭环 流程图 (你 写 手册 时 用 markdown 文字 流程 画)

```
[会前]                    [会中]                          [会后]
新建 会议 + 设议程  ──→  实时 转录 + AI 专家 发言  ──→  自动 抽 待办 / 经验 / 知识 草稿
邀请 AI 专家 进 会议      ↓                              ↓
                       智能 主持: 偏题 / 推进 /        审批 入库 (改一下 通过 / 弃用 / 退回)
                       决策 收口 / 时间 预警            ↓
                                                       AI 越来越 聪明 (下场 会议 自动 引用)
```

## 3.2 5 大 角色 体系 (政府 / 国企 多人 协作 必备)

| 角色 | 谁来当 | 能干啥 |
|------|--------|--------|
| **owner** | workspace 创建人 (一般 信息中心) | 总管, 可 配 系统 + 拆 团队 |
| **admin** | 信息部 + IT 主管 | 管 配置 + 审计 |
| **leader** | 部门 领导 / 处长 / 局长 | 召集 跨部门 会议, 推进 议程 |
| **manager** | 科长 / 业务 主管 | 维护 自己 主管 的 AI 专家 (审 它的 经验) |
| **member** | 普通 员工 | 进会议 + 提问 + 看决策 |

(每个 AI 专家 有 一个 manager — 它 抽 出 的 Memory / KB 草稿 由 manager 审, 跟 真 数字员工 一样)

- 审批 后 通知 草稿 来源

**会议 结束 沉淀**:

- 全景 时间线 (议程 进度 + AI 事件 时序)
