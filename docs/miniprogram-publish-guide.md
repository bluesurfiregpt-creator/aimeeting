# 智囊团 · 微信小程序上线操作指引

**专为运维 / 项目管理者使用**。本文档假设你**从来没操作过微信小程序后台**,把每一步从打开浏览器开始详细列出。照着做就能完成。

> 本文档不讲技术原理,只讲"打开什么、点哪里、填什么"。技术细节看 `docs/ops-test-guide.md`。

---

## 0. 先搞清楚:什么是"上线小程序"

简单说,一次完整的小程序上线 = **6 个阶段**:

```
准备   → 开发者工具上传 → mp 后台提交审核 → 等微信审核 → 发布 → 监控
~30min   ~10min            ~10min            1-3 工作日    几秒    24h
```

代码已经在 `wechat-miniprogram/` 目录里写完了,你**不需要写代码**,只负责把它"传到微信"并"提交审核"。

---

## 1. 准备工作(首次上线必做,第二次以后跳过)

### 1.1 你需要的账号 / 工具

| 项 | 准备 |
|---|---|
| 微信号 | 必须是**绑定到 AppID `wx1ed17af888224311` 的开发者微信号**;不知道是哪个就找产品负责人(bluesurfiregpt@gmail.com)|

### 1.3 拿到 `wechat-miniprogram/` 代码

产品负责人会把整个 `wechat-miniprogram/` 目录给你。**目录结构应该长这样**:

用任意文本编辑器(VS Code / 记事本 / TextEdit)打开:

- `wechat-miniprogram/project.config.json` → 找 `"appid"` → 应该是 `"wx1ed17af888224311"`(不是 `__YOUR_APPID__`)

如果不是,**找产品负责人**——他可能给错了版本。

## 2. 微信开发者工具:导入项目

### 2.1 打开微信开发者工具

如果是首次开:

1. 左侧菜单选 「**小程序**」
2. 右上点 「**+**」 按钮 → 出 弹框
3. 在弹框里:
   - **项目名称**:填 `智囊团`
   - **目录**:点右边的 📁 → 选你刚才放在电脑上的 `wechat-miniprogram` 文件夹 → 确定
   - **AppID**:应该自动从 project.config.json 读出来,显示 `wx1ed17af888224311`。如果没自动读出,手动填这个
   - **开发模式**:选 「**小程序**」
   - **后端服务**:选 「**不使用云服务**」
4. 点 「**确定**」 / 「**导入项目**」

### 2.2 验证导入成功

导入后会自动打开项目编辑器,你能看到:

- **左上**:模拟器(显示一个手机屏幕,默认加载 webview 进 `https://aimeeting.zhzjpt.cn/m`)
- **中间**:代码编辑区(可以不看)
- **右侧 / 底部**:调试器(可以不看)
- **右上**:`编译` / `预览` / `上传` 三个按钮

如果模拟器显示出**智囊团登录页**或一个空白页面 → 导入成功 ✅

如果显示**红色错误**(常见:`域名不在以下 list 中`)→ 域名没在 mp 后台配 → 找产品负责人 / 见本文档 5.1 / 但通常这步已经配过了

## 3. 测试小程序(必做,免得提审被打回)

### 3.1 在模拟器里跑一遍

在左侧模拟器里**手动操作**,跑通这几个场景:

| 场景 | 操作 | 期望 |
|---|---|---|
| 启动 | 打开默认就是 webview | 看到登录页 |
| 隐私同意 | 首次启动应弹"用户隐私指引"原生弹窗 | 点同意 |
| H5 登录 | 在 webview 里输 `demo.lijg@futian.gov.cn` / `demo123` | 跳到主页 |
| 主 tab 切换 | 点底部 今日 / 会议 / 任务 / 记忆 | 都能加载 |
| 创建会议 | 会议 tab → 右上 + → 进 `/m/meetings/new` | 看到创建表单 |

模拟器里**有问题就找产品负责人**——不要直接提审。

### 3.2 真机预览

模拟器跑通后,用真机扫一遍验证:

1. 开发者工具右上角点 「**预览**」(在"编译"旁边)
2. 等 10-30 秒 → 弹出二维码
3. 拿起手机,**用你那个开发者微信号扫码**(其他微信扫会提示无权限)
4. 微信打开「智囊团」小程序
5. 重复 3.1 几个场景,在真机里跑一遍

⚠️ 真机如果出现**模拟器没出现的问题**——必须解决,**不要带着 bug 提审**。

## 4. 上传代码到微信服务器

测试都过了 → 准备上传。

### 4.1 点"上传"按钮

开发者工具右上角点 「**上传**」 按钮(在"预览"旁边)。

### 4.2 填版本信息

弹框出现两个字段:

#### 字段 A:版本号

按规则填:

| 这次改动 | 版本号 |
|---|---|
| **首次提审** | `1.0.0` |
| **改了 bug** | 上次基础上 + 0.0.1,例 `1.0.1` |
| **加了新功能(不破坏老功能)** | 上次基础上 + 0.1.0,例 `1.1.0` |
| **彻底重写 / 不兼容老版** | 主版本 + 1,例 `2.0.0` |

不确定填什么 → 找产品负责人。

#### 字段 B:项目备注

写**这次改了什么**,审核员会看。**用中文短句**,例:

```
v1.0.1

新增功能:
- 新增原生会议室页 (录音 / 转录 / AI 流式 / 议程提醒)
- 新增原生创建会议页 (含 AI 拆议程 + 附件上传)

bug 修复:
- 议程时长选择改为原生 picker
- 修复 iOS 弹性滚动问题
```

如果是首次上线,可以这样写:

```
v1.0.0 首版

面向团队的 AI 协作会议工作台:
- AI 实时转录 + 议程监控
- AI 议程拆解 + 自主讨论
- 会议附件 + 微信聊天记录文件
- 任务派发 + 智囊洞察
- 长期记忆库

后端服务: https://aimeeting.zhzjpt.cn (已 ICP 备案)
隐私协议: https://aimeeting.zhzjpt.cn/m/privacy
```

### 4.3 点"上传"开始

点 「**上传**」 按钮 → 看进度条 → 等 30-90 秒。

成功时:
- 弹"上传成功"提示
- 关闭弹框

**这一步只是把代码传到微信服务器,用户还看不到**。下一步去 mp 后台提交审核才会让审核员看到。

## 5. mp 后台提交审核

### 5.1 登录 mp.weixin.qq.com

1. 打开浏览器 → [https://mp.weixin.qq.com](https://mp.weixin.qq.com)
2. **用你那个开发者微信号扫码登录**(就是开发者工具里登录的同一个)
3. 登录成功 → 看到一个深色后台

### 5.2 进入版本管理

1. 左侧菜单找 「**管理**」 → 点击展开
2. 子菜单点 「**版本管理**」
3. 进入版本管理页

### 5.3 找到刚上传的版本

页面分几个区:
- 「**开发版本**」← **看这个**
- 「**审核版本**」
- 「**线上版本**」
- 「**审核历史**」

在「开发版本」区,你应该看到**刚上传的版本号**(例 `1.0.1`)。

### 5.4 点"提交审核"

在你那个版本所在的行,**右侧操作列点 「提交审核」** 按钮。

会出现一个长表单(分多步)。逐步填:

#### Step 1: 选择类目

- 一般会预填好。**核对一下**:
  - 主类目:`商务服务 → 企业管理`
  - 副类目 1:`工具 → 效率`
  - 副类目 2:`商务服务 → 办公`
- 如果对就**点下一步**
- 如果想改 → 主类目改一次审核 5 个工作日,**慎重**

#### Step 2: 提供功能页面截图

要求:**每个主类目 1-3 张截图,展示对应功能**。

**怎么截图**(没准备过就这样做):

1. 回开发者工具,在模拟器里打开你想截的页面
2. **模拟器右上有截图按钮(相机图标),点它截屏**
3. 截到的图自动保存到电脑(通常 `~/Pictures` 或弹框选位置)
4. 准备 3-5 张,涵盖:
   - `/m`(今日 tab 主页面)
   - `/m/meetings/new`(创建会议页)
   - `/m/meetings/<某 id>`(会议室页面)

5. 回 mp 后台 → 点 「**上传图片**」 → 选刚截的图
6. 每张图要写**简短说明**(20 字内):
   - 截图 1:"主页 — 今日会议 + AI 智囊"
   - 截图 2:"创建会议 — AI 拆议程 + 上传资料"
   - 截图 3:"会议室 — 实时转录 + AI 协作"

#### Step 3: 测试帐号

填一个能让审核员登录的账号:

| 字段 | 填 |
|---|---|
| 测试帐号 | `demo.lijg@futian.gov.cn` |
| 测试密码 | `demo123` |
| 测试帐号附加说明 | `在 H5 webview 页面 输入邮箱密码 登录 即可` |

⚠️ **别填 owner 账号** `bluesurfiregpt@gmail.com`——权限太大给审核员用风险高。

#### Step 4: 上传演示视频(可选,但强烈推荐)

录 30-60 秒手机屏幕,展示一遍登录 → 创建会议 → 上传文件 → AI 拆议程 → 看智囊。

**审核员看了视频通过率会高 30%**。

录视频方法:
- iOS:控制中心 → 屏幕录制
- Android:下拉菜单 → 屏幕录制
- 模拟器:开发者工具支持录屏(右上的录制按钮)

录完上传到本步骤。

#### Step 5: 备注

填重要信息,帮审核员理解:

```
本小程序「智囊团」是 AI 协作会议工作台, 面向企业 / 政府 / 团队用户.
形态为 web-view 套壳 + 部分原生页 (会议室 / 创建会议 / 文件选择).
所有功能在 https://aimeeting.zhzjpt.cn 网站提供 — 业务域名 + 服务器域名 已配置 + 校验.

核心功能涉及:
1. 麦克风 — 仅在用户主动开启会议+点录音后采集, 退会即停
2. 文件选择 (wx.chooseMessageFile) — 让用户从微信聊天记录选会议参考资料
3. 用户隐私已在 H5 弹窗 + mp 后台隐私指引一致提交, 全文链接
   https://aimeeting.zhzjpt.cn/m/privacy
4. 第三方 SDK (阿里云通义千问 / 阿里云 OSS / pyannoteAI 等) 已在隐私指引完整列出

测试账号已给 leader 角色, 能看大部分功能 (会议/任务/智囊).
完整测试路径:
A. 登录后看 4 个 tab (今日 / 会议 / 任务 / 记忆)
B. 创建会议 + 让 AI 拆议程 + 上传 1 个 PDF
C. 进会议室看实时转录 / 召唤 AI 专家
D. 结束会议看总结页 (AI 纪要 + 待办)

如审核期间遇问题, 请联系 bluesurfiregpt@gmail.com.
```

#### Step 6: 加急审核(可选)

如果**真的很急**(例如 P0 hotfix),可以勾「**加急审核**」:

- 加急审核 = 4 小时内审完
- **1 年只有 5 次加急机会**,平时别用

### 5.5 提交

下方点 「**提交审核**」 按钮 → 弹"提交成功"。

回版本管理 → 「开发版本」区那行版本会消失 → 「**审核版本**」区出现该版本,状态 `审核中`。

## 6. 审核期间 等待

### 6.1 预计时长

- 普通审核:**1-3 个工作日**
- 加急审核:**4 小时内**

工作日 = 周一到周五,工作时间。**周末别提**,会卡到周一才审。

### 6.2 怎么知道结果

通过 / 被拒,你都会收到:

1. **微信通知**(那个开发者微信号会收到「公众平台安全助手」推送)
2. **邮件**(注册账号时的邮箱)

每隔几个小时你也可以**主动登录 mp 后台 → 版本管理 → 审核版本** 看状态。

### 6.3 审核期间 不要做什么

- ❌ **不要再上传新版本**——同时只能一个版本在审核中
- ❌ **不要修改 mp 后台 服务类目 / 名字 / 隐私指引**——可能影响审核结果
- ✅ 修改 H5 / 后端代码是 OK 的(那不在小程序审核范围内)

## 7. 审核结果

### 7.1 ✅ 审核通过

通知会写「**审核通过**」+ 「**点击发布**」链接。

你的版本会从 「审核版本」区 移到 可发布状态。**这一步审核员检查通过了,但还没让用户看到** —— 必须你**主动点发布**才上线。

跳到第 8 节 → 发布。

### 7.2 ❌ 审核被拒

通知会写**具体原因**,例如:

| 常见拒绝理由 | 怎么改 |
|---|---|
| `1.1 服务类目不准确` | 改类目(联系产品负责人确认要不要改主类目)|
| `5.1 测试账号不可用` | 自己用测试账号试登,看是不是密码改了 / 账号被禁 |
| `7.1 隐私不规范 — 隐私协议内容不一致` | H5 端 `/m/privacy` 全文页 跟 mp 后台填的"隐私指引"对不上 — 找产品负责人对齐 |
| `9.3 类目跟实际功能不符` | 提交时类目选错了 / 截图没体现该类目功能 |
| `没准备好` | 演示视频缺 / 截图模糊 / 备注太少 |

具体怎么应对:

1. 看清楚拒绝理由 → 找产品负责人讨论怎么改
2. 改完 → 回开发者工具 → 编译 → 测试 → **重新上传**(版本号 +1,例 `1.0.1` 改成 `1.0.2`)
3. 重新提交审核
4. 重新等审核

**被拒 不影响 mp 账号信誉**,可以无限次重提。

## 8. 发布(让用户拿到新版)

### 8.1 进入发布操作

审核通过后:

1. mp 后台 → **管理 → 版本管理**
2. 在「**审核版本**」区或上面的「**待发布**」区找到刚通过的版本
3. 点该行后面 「**发布**」 按钮

会弹一个选项:

| 选项 | 用途 |
|---|---|
| **全量发布** | 立即 100% 用户都能拿到新版 |
| **灰度发布** | 逐步放(5% → 20% → 50% → 100%),发现问题可以暂停 |

### 8.2 ⭐ 强烈推荐:首次上线用 灰度发布

灰度发布更安全。操作:

1. 选 「灰度发布」
2. **初始比例**:选 `5%`
3. 点确定 → 5% 用户开始拿到新版(你也是其中之一)

#### 灰度递增节奏(推荐)

**第 1 天**:
- 早上 10:00 → 灰度 5%
- 看 1 小时,后端日志没异常 → 全天保持 5%
- 你 / 测试 / 产品负责人 都用真机测一遍(用你那个开发者微信号一定在 5% 里)

**第 2 天**:
- 早上 → 看后端日志 / 用户反馈,没问题 → 灰度 20%
- 操作:版本管理 → 找在灰度的版本 → 点「**修改灰度比例**」 → 选 20%

**第 3 天**:
- 同上,加到 50%

**第 4 天**:
- 全量 100% — 点「**全量发布**」

灰度任何一步发现问题 → **暂停灰度** + 找产品负责人定位 + 必要时回滚(见 9.3)。

### 8.3 全量发布(图省事时用)

如果你信心十足 / 不是首次:
- 选「全量发布」
- 即时 100%

但**首次上线不建议直接全量** — 第一版总有意外。

## 9. 上线后 24 小时监控

### 9.1 怎么看小程序后台数据

mp 后台 → 左侧菜单 **数据 → 统计**

关注:
- **打开次数** — 24h 内有多少用户进来
- **访问人数** — 多少不重复用户
- **打开来源** — 微信搜索 / 分享 / 扫码 / 等
- **关键页面停留** — 用户在哪页停留长(说明这页很重要)

### 9.2 怎么看后端日志(找运维同事帮忙)

```bash
ssh root@47.245.92.62
docker logs aimeeting-backend --tail 500 -f
```

关注:
- `500 / Exception` 错误率 — 应 < 1%
- `WebSocket` 连接成功率 — 应 > 95%
- `LLM` 调用失败率 — 应 < 5%

### 9.3 怎么看用户反馈

mp 后台 → 左侧菜单 **用户反馈**

每个用户提的问题都在这里。**24 小时内回复一次**(简单一句话也行,让用户知道有人在管)。

### 9.4 ⚠️ 应急:发现问题怎么办

#### 小问题(界面 bug / 文案错)

不急,**改 H5** 立即生效(找产品负责人 / 开发者部署一下)。**不必小程序提审**。

#### 中等问题(原生页有问题但可以走 H5 绕过)

1. 找产品负责人改 H5 端把"原生入口按钮"关掉(`NATIVE_MEETING_ENABLED = false`)
2. 部署 H5(几分钟生效)
3. 用户继续用纯 H5,不再点到有问题的原生页
4. 等下个版本修好原生页再开

#### 大问题(整个小程序都跑不通)

紧急联系产品负责人,可能要:

1. **加急审核 hotfix** — 4 小时内审完一个修复版本
2. 或者 **回滚** — mp 后台 版本管理 → 找上一个稳定的"线上版本" → 点 「**回退**」(实际上是重新发布老版本)

回滚不是真的"撤销",微信不让真的撤销发布;只能"重新发上个版本"覆盖。

## 10. 第二次以后上线 简化流程

第二次上线时,1.x / 2.x / 5.x 都不用做了。只需要:

1. 拿到最新的 `wechat-miniprogram/` 代码(产品负责人给你)
2. 开发者工具 → 项目目录 同步代码(或者直接覆盖电脑上的旧目录)
3. 编译 + 真机测试(同 3.x)
4. 上传(同 4.x,**版本号要 +1**)
5. mp 后台 提审(同 5.x,**截图可以复用**)
6. 等审核 + 发布(同 6-8.x)

每次大概 1-2 小时操作 + 1-3 天等审核。

## 11. 常用术语对照表

第一次接触小程序后台容易懵的词:

| 词 | 含义 |
|---|---|
| AppID | 微信小程序的唯一标识,字符串 `wx1ed17af888224311` |
| 体验版 | 你扫开发者工具"预览"二维码看到的版本,仅开发者能看 |
| 开发版本 | 你刚上传到微信服务器但没提审的版本 |
| 审核版本 | 已提审,微信审核员正在看的版本 |
| 线上版本 | 已发布,所有用户看到的版本 |
| 灰度发布 | 不是 100% 用户看到新版,逐步放比例 |
| 业务域名 | web-view 加载 H5 时必须在这个白名单里 |
| 服务器域名 | wx.request / wx.uploadFile / wx.connectSocket 调用的目标域名 必须在这白名单 |
| MP_verify | 验证业务域名所有权的文件,放在 H5 服务器根目录 |
| 体验账号 | 给审核员登录用的账号,审核完成审核员会用它跑一遍主流程 |
