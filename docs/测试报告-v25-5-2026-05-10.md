### Bug #4 — F-4 · directive commit 后任务找不到【P1】

**现象**
- POST `/api/me/directives/{id}/commit` 返回 200
- response: `{"directive_id":"…", "committed_task_ids":["f44316ef-…"], "dispatched_count":0}`
- 用 `committed_task_ids[0]` 取任务详情:**两个端点都 404**

**复现**
```js
// leader 登录
const d = await fetch("/api/me/directives", {
  method:"POST", credentials:"include",
  headers:{"Content-Type":"application/json"},
  body: JSON.stringify({content: "请房屋安全科本周完成 3 栋 D 级危房应急避险措施"})
}).then(r=>r.json());
// d.id 拿到

const c = await fetch(`/api/me/directives/${d.id}/commit`, {
  method:"POST", credentials:"include",
  headers:{"Content-Type":"application/json"},
  body: JSON.stringify({tasks: [{
    content: "完成3栋D级危房应急避险措施",
    assignee_user_id: "1ecc73c6-0110-4b0b-937c-e87df9bfe04f", // 冯磊
    due_at: new Date(Date.now()+7*86400e3).toISOString()
  }]})
}).then(r=>r.json());
// c.committed_task_ids = ["f44316ef-…"], c.dispatched_count = 0  ❌

const t1 = await fetch(`/api/task/${c.committed_task_ids[0]}`,
                       {credentials:"include"}).then(r=>r.status);
// 404
const t2 = await fetch(`/api/tasks/${c.committed_task_ids[0]}`,
                       {credentials:"include"}).then(r=>r.status);
// 404
```

**期望**:
1. `dispatched_count: 1`(任务真发出去)
2. `GET /api/task/{id}` 200,status = `dispatched`,assignee_user_id = 冯磊
3. `/me` 「派发出」 tab 多 1 条

**怀疑位置**:commit 接口里写 task 但漏了 dispatched_at / assignee 字段;或者写到了不同的 db 表 / soft-delete 了。建议查 commit handler 内 ORM 写入和 dispatched_count 计算逻辑。

**衍生影响**:F-5(expert 签收)、F-6(AI 起草汇报)、F-7(leader 批准)、F-8(沉淀 KB)整条 v24/v25 任务闭环 demo 跑不通,客户演示前必须修。

### W-1 · A-2 seed 数量低于 spec
- `transcripts: 32` < spec 的 ≥50
- `kb_chunks: 48` = `kb_documents` 数,1 篇 1 chunk;spec 要 ≥80
- 怀疑:LLM chunker 对短文档(D-4 显示文档只有 191-214 字)直接当 1 chunk 返回。要么把 seed 文档加长,要么把 spec 阈值降到现实水平

- 页面渲染了 24 个成员,但右下角红 toast `请求失败 (403) owner or admin required`
- 应该:leader 进 admin 页直接 redirect 或 403 整页,而不是渲染了又来个 toast
