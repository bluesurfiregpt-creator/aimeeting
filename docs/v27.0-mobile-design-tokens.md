# v27.0-mobile · 设计 token (字号 / 间距 / 颜色 / 触摸)

定 token 的目的: 锁死「不许再用 web 字号」这条线. 任何 mobile 组件
(`frontend/src/app/m/**`, `frontend/src/components/mobile/**`) 落代码前必查此表.

参考: iOS HIG (Human Interface Guidelines) + Material Design 3, 取交集.

## 1. 字号 (Tailwind className)

| Token | className | px | 字重 | 用途 |
|-------|-----------|----|------|------|
| display | `text-[28px] font-semibold` | 28 | 600 | 页头 H1 ("今日" / "会议" / "任务") |
| title-lg | `text-[20px] font-semibold` | 20 | 600 | 卡片大标题 (Hero 内会议名) |
| title-md | `text-[17px] font-semibold` | 17 | 600 | 卡片主标题 / 段落标题 |
| body | `text-[16px]` | 16 | 400 | 默认正文 |
| body-sm | `text-[15px]` | 15 | 400 | 紧凑正文 / 列表行内容 |
| label | `text-[14px] font-medium` | 14 | 500 | 按钮 / chip / tab / 小卡标签 |
| caption | `text-[13px]` | 13 | 400 | **最小可读下限**. 辅助说明 / 元信息 |
| num-lg | `text-[20px] font-semibold tabular-nums` | 20 | 600 | 大数字 (任务数 / 计数) |
| num-md | `text-[16px] font-medium tabular-nums` | 16 | 500 | 中数字 (chip 内数字) |

### 禁用
- ❌ `text-[10px]` / `text-[11px]` / `text-[12px]` — 不允许作为任何可读内容. 12px 在 6 寸屏 + 实际握持距离下已模糊.
- ❌ `font-light` / `text-xs` (Tailwind 默认 12px) 用作 body.

### 例外
- 角标 (e.g. 红点未读数 "9+") 可以用 `text-[11px]`, 但仅限非阅读性视觉指示.

## 2. 间距

| 用途 | 值 | className |
|------|----|-----------|
| 页边距 | 16px | `px-4` |
| 卡片之间 | 12-16px | `space-y-3` / `space-y-4` |
| 卡内 padding | **16px (最小)** | `p-4` |
| 段落内行距 | 8px | `space-y-2` |
| chip 与文字 | 8px | `gap-2` |

### 禁用
- ❌ `p-2` / `p-2.5` / `p-3` 用于可点击卡片 — 触摸热区不够.
- ❌ `space-y-1` (4px) 用于卡间.

## 3. 颜色 (深色主题, zinc-* 体系)

| 用途 | 值 | 说明 |
|------|----|------|
| 主标题 | `text-zinc-50` | 最重要 |
| 主体正文 | `text-zinc-100` 或 `text-zinc-200` | 默认可读 |
| 次文字 / caption | `text-zinc-400` | 解释/元信息 |
| dim / 装饰 | `text-zinc-500` | 时间戳 / 分割符 |
| 强调蓝 | `text-accent-400` / `text-accent-300` | 链接 / 主操作 |
| 警示橙 | `text-amber-300/90` | 超时 / 临界 |
| 危险红 | `text-rose-300` | 超期 / 错误 |
| 成功绿 | `text-emerald-300` | 通过 / 完成 |

### 禁用
- ❌ `text-zinc-600` / `text-zinc-700` 作为可读文字 — 对比度不足. 仅可做边框 (`border-zinc-700`).

## 4. 触摸热区

| 元素 | 最小高 | className |
|------|--------|-----------|
| 主按钮 | 48px | `h-12` |
| 列表行 (可点) | 56px | `min-h-14` |
| 卡片 (可点) | 80px+ | 自然布局 + `p-4` 即可达到 |
| chip (可点) | 32px | `h-8 px-3` |
| icon-only 按钮 | 44px | `h-11 w-11` |

## 5. 字号决策捷径

写字段时按"信息重要度"问自己:
1. 这是用户要找的核心信息吗? → `title-md` (17px)
2. 这是说明这条信息是什么的标签? → `label` (14px)
3. 这是辅助元数据 (时间/计数/源)? → `caption` (13px)
4. 想 < 13px? → 没有此选项. 重新设计信息层级.

grep -rn 'text-\[1[012]px\]\|text-xs' frontend/src/app/m/ frontend/src/components/mobile/

## 7. 适用范围

仅限 `/m/*` 路由 + `components/mobile/*` 目录. 桌面端不变.
