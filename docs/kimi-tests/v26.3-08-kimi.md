## 3. 测试用例 T-01 ~ T-06

### T-01 · 新会议 → state 里有 `running_elapsed_sec` + `max_meeting_sec` 字段

**前置**:P-1/P-2/P-3 全 pass
**目标**:验证 v26.3-08 后端 schema 已经生效

**步骤**:
1. 在首页创建 auto 会议:议程 2 项(任意标题)、勾 3 个 expert、标题 `kimi-test-08-T01-<时间戳>`
2. 跳到 `/meeting/<id>/orchestrate` 后,**不要** 点启动
3. 浏览器 F12 → Network → 找最近一次 `GET /api/meetings/<id>/orchestrate/state`
4. 看 response body

**Pass 条件**(全部):
- response body 是 JSON,**包含字段** `running_elapsed_sec` (字面值是 0 或浮点数,number 类型)
- response body **包含字段** `max_meeting_sec` (字面等于 `2700`,即 45*60)
- response body **包含字段** `phase`(字面等于 `"idle"`)

**Fail 条件**:
- response 没有 `running_elapsed_sec` 字段 → fail("v26.3-08 后端 schema 未生效")
- `max_meeting_sec` 不是 2700 → fail(配置错或路由没重启)

**必须复述**:把完整 response JSON 粘贴到报告(只这一次,不要省略 任何字段)

### T-02 · idle 时前端 不显示 已用时间

**前置**:T-01 pass,还在 idle 控制台页
**步骤**:
1. 看页面右上角 phase 徽章 (`未启动 / idle`)旁边
2. 寻找 `⏱` 时间显示

**Pass 条件**:
- idle 状态 **没有** `⏱` 时间显示(因为还没启动)
- 只显示 phase 徽章 + `0 / 2 议程完成`

**Fail 条件**:
- idle 状态 看到 `⏱ 0:00 / 45:00` → fail("idle 不该显示 计时器")

**必须复述**:截图整个页面顶部右侧

### T-03 · 启动 → 前端开始显示已用时间

**前置**:T-02 pass
**步骤**:
1. 点 🚀 启动 AI 自主讨论
2. 等 8 秒
3. 截图整个页面顶部右侧

**Pass 条件**(全部):
- phase 徽章 变 `运行中 / running`
- 旁边出现 `⏱ 0:0X / 45:00` (X 是 1-9 数字)
- API 验证(浏览器 cookie):

  - `phase` 字段 字面等于 `"running"`

**Fail 条件**:
- `⏱` 没出现 → fail("前端 timer 渲染失效")
- `running_elapsed_sec` ≥ 30 → fail("计时器异常快")

### T-04 · 暂停 → 计时器 冻结 (Q8=B 决策落地)

**前置**:T-03 pass,phase=running 且 `running_elapsed_sec` 已涨到 X1 秒
**步骤**:
1. 记下当前 `running_elapsed_sec` = X1 (从浏览器 F12 Network 看最近一次轮询)
2. 点 ⏸️ 暂停
3. 等 **15 秒**(壁钟时间)
4. 看新一次 `GET /orchestrate/state` 的 `running_elapsed_sec` = X2
5. 看页面右上角 `⏱` 数字 在这 15 秒内 **应该几乎不动**

**Pass 条件**(全部):
- X2 - X1 字面值 **< 3 秒**(只允许 0-3 秒的浮动,因 暂停信号 + 议程结束有 LLM 飞行时间;**绝不允许 X2 - X1 > 5**)
- 页面 `⏱` 数字在 15 秒内 数字不动(目测)
- `phase` 字段 字面等于 `"paused"`
- `paused_at` 字段 字面 是 ISO 字符串(非 null)

**Fail 条件**:
- X2 - X1 > 5 秒 → fail("paused 期间继续累计,Q8 决策未生效")
- `phase` 不是 `"paused"` → fail("暂停未生效")

**必须复述**:
- X1 数值 + X2 数值 + 差
- 完整 paused 时的 state JSON

### T-05 · 恢复 → 计时器从冻结值继续 (不归零)

**前置**:T-04 pass,phase=paused 计时器停在 X2
**步骤**:
1. 点 ▶️ 恢复
2. 等 6 秒
3. 看新 `running_elapsed_sec` = X3

**Pass 条件**:
- `phase` 字面等于 `"running"`
- X3 字面值 **> X2** 且 **< X2 + 10**(从冻结点继续,不归零,且累计 ≈ 6 秒)
- `paused_at` 字段 字面等于 `null`

**Fail 条件**:
- X3 ≈ 0 → fail("恢复后归零,paused_running_seconds 没保留")
- X3 < X2 → fail("时间回退,逻辑错")
- X3 > X2 + 15 → fail("把 paused 那 15 秒补上了,Q8 决策错误")

**必须复述**:X2 / X3 / 差
