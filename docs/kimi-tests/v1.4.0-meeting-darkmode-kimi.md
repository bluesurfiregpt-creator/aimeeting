## 3 · 测试用例 T-01 ~ T-08

### T-01 · default 浅色 (零回归)

**前置**: P-3 pass

**目标**: 验证 会议室 首次访问 (storage 空) 默认 浅色

**步骤**:
1. 开 浏览器 DevTools → Application → Local Storage → 清 `w-theme` key
2. 硬刷 `https://aimeeting.zhzjpt.cn/meeting/q3-roadmap/live`
3. 在 Elements panel 看 `<html>` 的 `data-theme` attr
4. Computed style 查 `body` 或 `MRLiveView` 根 div 的 `background-color`

**Pass 条件** (字面):
- `<html data-theme="light">` (字面 "light")
- 计算后的 bg 是 `rgb(242, 242, 247)` (= `#F2F2F7` light bgChip)
- TopBar 背景 计算后 是 `rgb(255, 255, 255)` (= `#fff` light bgTopBar)
- 文字主色 计算后 是 `rgb(28, 28, 30)` (= `#1C1C1E` light fgPrimary)

**Fail 条件**:
- `data-theme="dark"` (storage 空时不应是 dark)
- bg 是 `rgb(5, 7, 26)` (= `#05071A` dark bgCanvas — 不该出现在浅色)

**必须复述** (Kimi 验收时):
- 截图 顶 nav + 三栏布局
- DOM `<html data-theme="...">` 完整 attr 字符串
- bg / TopBar / 文字主色 三个 computed `rgb(...)` 值, 不要缩写

### T-02 · 主题切换按钮 可见

**前置**: T-01 pass

**目标**: 顶 nav 看到 MRThemeToggle segmented control

**步骤**:
1. 在 会议室 live 页面 顶 nav (Row 1) 找到 "浅" / "深" segmented control
2. 截图 整 Row 1

**Pass 条件**:
- DOM 看到 `<div role="group" aria-label="主题切换">`
- 内含 2 个 `<button>`, 一个 字面 "浅" (active), 一个 字面 "深" (inactive)
- "浅" 按钮 `aria-pressed="true"`, "深" 按钮 `aria-pressed="false"`
- 位置 在 "设置" 按钮 跟 "结束会议" 红色按钮 之间

**Fail 条件**:
- 没看到 toggle (PR 集成失败)
- "深" active 而 "浅" 不 active (storage 漏读 / bootstrap 漏跑)
- toggle 出现 在 顶 nav 之外 (e.g. 工作站 — 不该出现)

**必须复述**:
- 整 Row 1 截图
- 两个 button 的 `aria-pressed` 值 (完整 HTML attr 字符串)

### T-03 · 切深色 → CSS var 翻转

**前置**: T-02 pass

**目标**: 点 "深" → data-theme 变 dark, CSS var 翻转, 视觉 转 深邃星空

**步骤**:
1. 点 "深" 按钮
2. 立刻 截图 全 viewport
3. DevTools Elements 看 `<html data-theme>` attr
4. Computed style 查 `MRLiveView` 根 div bg, TopBar bg, 主文字

**Pass 条件** (字面):
- `<html data-theme="dark">` (字面 "dark")
- "深" 按钮 现在 `aria-pressed="true"`, "浅" 现在 `aria-pressed="false"`
- bg 计算后 是 紫色 tint (e.g. `rgba(124, 92, 250, 0.1)` 或类似)
- TopBar bg 是 linear-gradient (含 `rgb(11, 15, 38)` 跟 `rgb(8, 11, 31)`)
- 主文字 是 `rgb(245, 245, 247)` (= `#F5F5F7` dark fgPrimary)

**Fail 条件**:
- 切完 还看到 白底 (CSS var 没翻)
- 切完 还看到 黑字 (fg 没翻)
- 切完 toggle 状态没变 (event 没绑)

**必须复述**:
- 切换前后 两张 截图
- `<html data-theme>` 切换前 vs 切换后 (字面 light / dark)
- 至少 3 个 computed `rgb(...)` (bg / TopBar / 主文字), 不要省

### T-04 · 深色 关键 hex 校对 (设计稿对齐)

**前置**: T-03 pass

**目标**: 深色 模式 关键 颜色 跟 设计稿 S3TK 严格对齐

**步骤**:
1. 在 dark mode 下, DevTools Elements 看 `<html>` 的 inline / inherited CSS var
2. 在 Computed panel 找 `--mr-bg-canvas`, `--mr-accent-playhead`, `--mr-fg-primary` 三个 var

**Pass 条件**:
- `--mr-bg-canvas` 值 字面 `#05071A` (或 `rgb(5, 7, 26)`)
- `--mr-accent-playhead` 值 字面 `#B9A0FF` (紫色 active speaker 光带)
- `--mr-fg-primary` 值 字面 `#F5F5F7`
- 若 `--mr-bg-stage` 存在, 值 含 `linear-gradient` 且 stop 含 `#060818` / `#0A0E22`

**Fail 条件**:
- 任一 var 值 跟 上述 不符 (设计稿 漂移)
- var 在 dark mode 下 仍是 light value (CSS 没 scope 到 `:root[data-theme="dark"]`)

**必须复述**:
- 4 个 CSS var 的 完整 computed value (含 hex / gradient 完整字符串)

### T-05 · localStorage 持久化

**前置**: T-03 pass (当前 dark mode)

**目标**: 关 tab 重开 → dark mode 保留 (storage 持久)

**步骤**:
1. DevTools Application → Local Storage → 查 `w-theme` key
2. 关掉 tab
3. 重新 navigate 到 `https://aimeeting.zhzjpt.cn/meeting/q3-roadmap/live`

**Pass 条件**:
- storage `w-theme` 值 字面 `"dark"`
- 重开 后 第一时间 (hydrate 前) `<html data-theme="dark">` 已设好 (bootstrap script)
- 视觉 仍 深邃 (无 浅 → 深 flash)

**Fail 条件**:
- storage 没存 (toggle 没写 localStorage)
- 重开 后 仍 light (bootstrap 没读 storage)
- 重开 后 有 闪烁 (bootstrap script 跑慢于 hydrate)

**必须复述**:
- storage key/value 截图
- 重开后 第一帧 截图 + `<html data-theme>` 字符串

### T-08 · reduced-motion (可选)

**前置**: T-07 pass

**目标**: 用户 启用 系统级 `prefers-reduced-motion` 时, dark mode 的 aurora / starfield
动画 应 不渲染 (保 静态 视觉)

**步骤**:
1. DevTools → ⋮ → More tools → Rendering → "Emulate CSS media feature
   prefers-reduced-motion" → 选 `reduce`
2. 切 dark mode
3. 看 stage 中央 (transcript 区) 是否 有 aurora 流动 / starfield 闪烁

**Pass 条件**:
- 无 aurora 流动 动画 (静态 渐变 ok)
- 无 starfield 闪烁
- transcript 内容 正常 渲染, 无 jitter

**Fail 条件**:
- aurora 仍 在 流动 (违反 § 7.1.1 第 4 条)
- starfield 仍 闪烁

**注**: 若 v1.4.0 本版 还没 实施 aurora/starfield ambient, 此用例 标 SKIP (不算 fail).
