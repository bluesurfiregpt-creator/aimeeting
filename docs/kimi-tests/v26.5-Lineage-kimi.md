## 测试用例

### A. Memory 多对多

#### T-01 · MGR1 写 memory 归 AGENT_1, 同时挂 AGENT_2 → 200, 多 agent 显示

**步骤**:
- MGR1 进 `/me/profile/memory`
- form 选 agent_id = AGENT_1 (注: 当前 P1 UI 仍是单 select; 多 agent 走 curl 测)
- 或 直接 curl POST `/api/memory` body `{scope:"project", content:"T01", agent_ids:[AGENT_1, AGENT_2]}`

**Pass**:
- 返回 200, memory.agents 数组含 2 个 entry
- AGENT_1 is_primary=TRUE, AGENT_2 is_primary=FALSE
- 列表显示 ⭐ AGENT_1 + 🔗 AGENT_2 两个徽章

#### T-02 · MGR1 删 自己写的 memory → 200
- DELETE `/api/memory/{id}` 200

#### T-03 · MGR2 删 MGR1 写的 memory → 403
- MGR2 不是 AGENT_1 / AGENT_2 的 primary → 403 `[权限不足]`

#### T-04 · MGR1 按 agent_id 过滤 memory list
- GET `/api/memory?agent_id={AGENT_1}` → 返回所有 挂在 AGENT_1 (无论 primary 还是 subscriber) 的 memory

### B. Memory 审批 gate (会议来的)

#### T-05 · 创建 task → MGR2 (非 AGENT_1.primary) 办结 → MemoryDraft 生成

**前置**: AGENT_1.primary = MGR1, AGENT_1 已有 KB

**步骤**:
1. OWNER 派 task 给 AGENT_1, 让 MGR2 协办
2. MGR2 办结 → owner approve
3. 等 closure_curator 跑 (几秒)
4. GET `/api/memory-drafts` (MGR1 视角) — 应该看到 1 个 pending draft
5. 同时 `GET /api/memory` 直接查 LongTermMemory — **没有新 memory** (走 gate 了, 还没写)

**Pass**:
- draft.status = pending
- draft.target_agent_ids = [AGENT_1]
- draft.source_type = "task"
- draft.source_task_id = 该 task
- LongTermMemory 表无新行
- MGR1 顶栏 / `/me/profile` 看到红 badge

#### T-06 · MGR1 进 /me/profile/sedimentation → 切到 🧠 Memory 沉淀 tab

**Pass**:
- 顶部 2 个 tab: 📚 KB 沉淀 / 🧠 Memory 沉淀
- Memory 沉淀 tab 红 badge 显示 "1"
- 列表显示 T-05 创建的 draft

#### T-07 · MGR1 点击 查看 → 批准 → 真正写 long_term_memory

**Pass**:
- POST `/api/memory-drafts/{id}/approve` 200
- draft.status = approved, committed_memory_id 指向新 memory
- GET `/api/memory?agent_id={AGENT_1}` → 看到这条 memory, agents=[{AGENT_1, primary=TRUE}]
- memory.source_meeting_id 应该有值 (如果 task 来自会议) 或 NULL

#### T-08 · 再来一次 — 这次 MGR1 驳回

**Pass**:
- POST `/api/memory-drafts/{id}/reject` body `{reason:"内容不准确"}` 200
- draft.status = rejected, decision_reason="内容不准确"
- LongTermMemory 表 没新行
- MGR2 收到 通知

#### T-09 · MGR1 自己办的 task → 不走 gate, 直接写

**前置**: AGENT_1.primary = MGR1

**步骤**:
- OWNER 派 task 给 AGENT_1, 让 MGR1 (= primary) 自己办结

**Pass**:
- closure_curator 直接写 long_term_memory (老路径)
- 不创建 MemoryDraft
- GET `/api/memory-drafts` 应该不见这个 task 的 draft
