## 顶部 6 条 死规矩 (反幻觉)

1. **真实页面 / HTTP** — 每步 Pass/Fail 基于 浏览器 实际 看到 的 DOM 或 curl 输出
2. **每步 独立 判定** — 不允许 "按理说应该..." / "上一步过了 这一步也应该过"
3. **复述 原文** — 引用 必须 带 引号 + 是 实际渲染出来 的 字面字符 (不允许 翻译/总结)
4. **不许 跳步** — fail 则 后续 BLOCKED
5. **不许 编造账号** — 仅 §账号 表 5 个账号
6. **每用例 4 段** — (1) 实际看到 (2) 判定 pass/fail/blocked (3) 失败 理由 (4) 证据 (截图 + JSON 原文)
