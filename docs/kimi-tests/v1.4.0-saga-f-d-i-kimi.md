## 顶部 6 条死规矩(反幻觉,不可忽略)

1. **只看真实页面 / 真实 HTTP 响应**: 任何 "应该" / "估计" / "通常" 的判定全部不接收。
2. **每步判定必须基于本步实际看到的数据**: 不允许把上一步的 DOM 拿来给这一步用。
3. **复述要原文**: 带引号的精确字符串 (按钮文字 / chip 文字 / 颜色 hex / API 字段); 不允许概括。
4. **不允许跳步**: 前一步 fail 则后续 BLOCKED, 不允许 "假设过了" 继续。
5. **不允许编造账号**: 只用本文档 `## 环境与账号` 段的 5 个邮箱+密码字面常量, 其它一律拒。
6. **每用例必须 4 段**: 实际看到 / 判定 / 失败理由 / 证据 (4 段缺 1 段, 整 case 判 fail)。

## 预检 P-1 ~ P-4 (任一 fail → 所有 T-* BLOCKED)

### P-1: 服务器健康
- **目标**: 站点活
- **步骤**: `curl -sS -o /dev/null -w "%{http_code}" https://aimeeting.zhzjpt.cn/`
- **Pass**: 输出 `200`
- **Fail**: 任何 5xx / timeout / DNS error
- **复述**: 完整 HTTP 状态码

### P-2: 后端 API 活
- **目标**: backend container 接口可访问
- **步骤**: `curl -sS -o /dev/null -w "%{http_code}" https://aimeeting.zhzjpt.cn/api/auth/login`
- **Pass**: 输出 `405` (GET 不允许, 但 endpoint 存在)
- **Fail**: 502 / 503 / 504 / timeout
- **复述**: HTTP 状态码

### P-3: H5 根页 活
- **步骤**: `curl -sS -o /dev/null -w "%{http_code}" https://aimeeting.zhzjpt.cn/m`
- **Pass**: `200` (是 H5 SPA 入口)
- **复述**: HTTP 状态码

### P-4: 登录后台口子
- **步骤**: 用 `demo.lijg@futian.gov.cn` / `demo123` 走 `POST /api/auth/login`
  ```
  curl -sS -X POST https://aimeeting.zhzjpt.cn/api/auth/login \
    -H "Content-Type: application/json" \
    -d '{"email":"demo.lijg@futian.gov.cn","password":"demo123"}'
  ```
- **Pass**: JSON 含 `access_token` 字段 (非空 string)
- **Fail**: 401 / 403 / 5xx / 字段缺失
- **复述**: 完整 JSON 原文 (不省略字段)
- **保存**: 把 token 存为 `LEADER_TOKEN` 给后续 T-* 用

## T-01: H5 二级页 浅色 — `/m/meetings/new`

- **前置**: P-1 ~ P-4 全 pass
- **目标**: 创建会议页 已浅色化 (跟 /m today 一致)
- **步骤**:
  1. 浏览器装载 `https://aimeeting.zhzjpt.cn/m/meetings/new` (或先登录拿 cookie 再访问)
  2. 截屏整页
  3. 用 DevTools 检查 `<body>` 或 root section 的 `background-color` 计算值
  4. 检查 h1 / 主文字的 `color` 计算值
- **Pass 条件** (字面值):
  - 页面背景 `rgb(242, 242, 247)` 或更接近 (=`#F2F2F7`, iOS 系统灰)
  - 主文字 `rgb(28, 28, 30)` 或更接近 (=`#1C1C1E`)
  - **不应** 出现 `#0a0a0c` (老 dark) / `rgb(10, 10, 12)` / `rgba(0,0,0,*)` 占主背景
- **Fail 条件**:
  - 页面整体 暗紫 / 黑 / 任何 `#0a0a0c` 主背景 → fail
  - 文字 `#fff` / `#e5e5e5` 主色 → fail (老 dark 残留)
  - 卡片 / 输入框 dark 描边 → fail
- **必须复述**: 整页截图 + DevTools 计算样式 (`background-color` / `color` 两个字段原文)

  1. **leader** (`demo.lijg`): 登录 → `/m/me` 截屏 → 找 link `data-testid="mobile-me-workstation-link"` → 抓其 `href` 属性
  2. **admin** (`demo.chensy`): 同上
  3. **agent_owner** (`demo.fengl`): 同上, 但应**不见** 该元素
  4. **member** (`demo.hanx`): 同上, 应**不见**
- **Pass**:
  - leader + admin: 看到入口, `href` 原文为 `/workstation` (或绝对 URL `https://aimeeting.zhzjpt.cn/workstation`)
  - agent_owner + member: DOM 中查不到 `data-testid="mobile-me-workstation-link"`
- **Fail 条件**:
  - leader / admin 看不到入口 → fail
  - href 是 `/admin` (老路径) → fail (这就是 Saga I 要修的)
  - agent_owner / member 看到入口 → fail (越权)
