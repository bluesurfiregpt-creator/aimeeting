## A. AI 模板生成器 (核心)

### T-01 · 入口可见
**步骤**: owner 登录 → /me/profile → sidebar 点 `✨ AI 模板生成器`
**Pass**:
- URL = `/me/profile/agents/template`
- 看到 "✨ AI 模板生成器" + 输入框 + 数量 1-10 + 两个 checkbox (种子 KB + 种子 Memory)
- 不再是 "🚧 即将上线" 占位

### T-02 · manager / member 视角拒
**步骤**: manager 访问 /me/profile/agents/template
**Pass**: 显示 "⛔ 权限不足" + 提示 "仅 owner / admin / leader 可用"

### T-03 · 生成预览 (核心)
**步骤**:
1. owner 进 模板生成器
2. 在 textarea 输入: "我是一家科技公司,要 5 个 AI 助手 — 产品/技术/法务/财务/HR"
3. 数量 = 5
4. 勾选 "📚 生成种子知识库" + "🧠 生成种子长期记忆"
5. 点 "✨ 生成预览"
6. 等 LLM 调 ~10-30 秒

**Pass**:
- 按钮变 "⏳ 生成中…"
- 完成后 跳到 step 2 预览页, 显示 5 个 agent 卡片
- 每个卡片:
  - 名字 (例: "产品策划专家")
  - 领域 (例: "产品")
  - 颜色 dot (6 种不重复)
  - 展开 ↓ 看到 persona / keywords / 种子 KB markdown / 种子 Memory list

**Fail**: 503 (LLM 未配置) / 解析失败 / 空数组

### T-04 · 编辑 draft
**步骤**: 点某个卡片右边 "↓" 展开 → 改 name / persona → 改颜色
**Pass**: 内容跟着变 (受控 input)

### T-05 · 删除 draft
**步骤**: 点 ✕
**Pass**: 该卡片消失,列表数量减 1

### T-06 · 手动添加空白
**步骤**: 点 "+ 手动添加一个空白 AI"
**Pass**: 列表加 1 个空 draft (用户可填)

### T-07 · 选 manager 分配 (round-robin)
**步骤**: 在 "🎯 自动分配 primary_user" 区 勾选 2 个 manager
**Pass**:
- 勾的 manager 卡片变 violet 高亮
- commit 后 第 1 个 agent 的 primary = 第 1 个勾的 manager, 以此类推

### T-08 · 一键创建 (核心)
**步骤**: 点 "🚀 一键创建 N 个 AI"
**Pass**:
- 按钮变 "⏳ 创建中…"
- 完成 toast `✅ 已创建 N 个 AI 专家`
- 1.2 秒后 自动跳 `/me/profile/agents`
- AI 列表里 看到 5 个新建的 agent (含颜色 dot + name + 领域 + 主科室)
- 进 KB 列表 → 5 个种子 KB (名 "<agent>·种子知识库", owner_agent 已绑)
- 进 Memory 列表 → N 条种子 memory (每条 ⭐ 挂对应 agent)
- 进 全景血缘图 → 5 个新节点 + 来源 (📁 owner 名) → KB + Memory → AI

### T-09 · 后端 curl preview
```bash
curl -b cookie.txt -H "Content-Type: application/json" \
  https://aimeeting.zhzjpt.cn/api/agent-templates/preview \
  -d '{"scenario_description":"测试", "count":2, "with_kb_seed":false, "with_memory_seed":false}'
```
**Pass**: 返回 `{agents: [{name, domain, persona, keywords, color}, ...], scenario_description, raw_llm_text}`

## D. 沉淀草稿过期 sweep

### T-13 · 模拟过期 (人工 SQL)
**前置**: 改一个 pending KbSedimentationDraft 的 `created_at` 为 8 天前
```sql
UPDATE kb_sedimentation_draft SET created_at = NOW() - INTERVAL '8 days'
WHERE id = '<some-pending-id>';
```
**步骤**: 等 1 小时 / 或 重启 backend
**Pass**: draft.status 从 `pending` 变 `expired`

(可选) 加 logger.info 验证: backend log 含 `[draft_expire_sweeper] expired X KB drafts, Y Memory drafts`
