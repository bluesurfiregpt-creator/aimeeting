- **前置**: P-1 ~ P-5 全 pass
- **目标**: 校验 `frontend/src/components/mobile/meeting-room/mock/roundtable.ts` 里 写死的 mock 字符串 (eg `"把 B 组灰度到 20%, 可推进吗?"`, `"ARIA"`, `"SAGE"`, `"LEX"`, `"+11.4pp"`) 已彻底从生产 bundle 里 删除.

  3. 进任一 meeting 详情页 → 滚到底

  - 所有 JS chunk **找不到** 字符串 `"+11.4pp"`

### T-02 · /m/meetings/{id} 聚合 API 真返回 attending_agents (Item 1 验证后端真接还活着)

- **前置**: P-* 全 pass + T-01 拿到 meeting UUID
- **目标**: 校验 backend `/api/m/meetings/{id}` 真返回 agent 列表 (不是 mock).
- **步骤**:
  ```bash
  curl -sS https://aimeeting.zhzjpt.cn/api/m/meetings/<MEETING_UUID> \
    -H "Authorization: Bearer $TOKEN" | jq '.attending_agents | length, .mode, (.attending_agents[0] | {agent_id, name, role})'
  ```
- **Pass 条件**:
  - `attending_agents` 长度 ≥ 1 (整数)
  - `mode` 字段 ∈ {`"auto"`, `"hybrid"`, `"human"`}
  - 第一个 agent 含 `agent_id` (UUID), `name` (非空字符串), `role` (字符串)
- **Fail 条件**:
  - `attending_agents` 长度 = 0 → 真数据没 seed 进
  - 字段缺 `agent_id` / `name` → 字段 schema 漂移
  - HTTP 5xx → 后端 crash
- **必须复述**: jq 输出原文 (含数字 + JSON object)

## 风险 + PM 后置 manual 测试 提示

- **风险 1** (回归): hybrid mode 不再 显 demo 卡, 如果 Item 2 阈值/prompt 没让真 AI 触发,
  会议时间内 timeline 可能 显空白 (空 transcript + 没 AI 主动 chime in).
  → PM 实测: 打开一场 hybrid meeting, 说几句话 (任何包含 "数据 / 法务 / UX" 关键词),
  看 4-15s 内 是否 出现 真 AI streaming 发言.

- **风险 2** (阈值过激): 4s/15s 可能 触发 AI 频繁 插话 spam.
  → PM 实测: 长会议 (10-15min) 看 AI 发言频率是否 合理 (~1-2 min 1 次, 不是 4s 1 次).

- **风险 3** (立场过激): "禁止和稀泥" prompt 可能 导致 AI 在不该选边的中性问题上 强行 站队.
  → PM 实测: 问一个 中性问题 (eg "今天 天气 不错"), AI 回应是否 自然 (没强行 拉
  到 立场 / 推荐).
