# v26.13.1 Kimi 用例 · AI 私聊 调试模式

> 版本: v26.13.1 (私聊 真实现, 替代 v26.12 mock)
> 范围:
>   - **POST `/api/agents/{id}/chat`** SSE 流式 endpoint — 复用 `_call_dify_and_stream`-style 路径, 完全 read KB + memory, 不写 transcript/memory/KB
>   - **POST `/api/chat/parse-file`** in-memory PDF/docx/image OCR (≤ 20MB)
>   - **WS `/ws/chat-stt`** 语音输入 (FunASR), 不写 transcript
>   - per-user 日配额 50 次/天
>   - 前端 `/chat/[id]` 替代 mock — 消息 列表 + 文件上传 + 语音输入 + AI 底牌 sidebar + sessionStorage 持久
>   - 调试模式 顶部 大字 提示 "🔧 内容不保留"
> 测试者: Kimi
> 预计时长: 25 分钟

## T-01 · 进入 私聊页 + 顶部 调试模式 提示 ★

**前置**: P-3 pass; 在 `/` 上 找到 任意 AI 卡片.

**步骤**:
1. 点 任一 AI 卡片 的 "📣 召唤" 按钮
2. 召唤 modal 弹出, 点 "💬 一对一 私聊"
3. 跳转 至 `/chat/<agent_id>`

**Pass** (必须 全部):
- (a) 顶部 有 一条 amber 色 banner, 包含 文字 `"🔧 调试模式 · 内容不保留 (sessionStorage)"` (字面)
- (b) banner 右侧 显示 `"今日剩余 N 次"` (N 是 数字; 首次 进入 可能 还没显示, 发完 一次消息 后 必定 显示)
- (c) 中间 是 一个 主对话区, 上方 显 AI 头像 + nickname (或 name) + domain
- (d) 主对话区 默认 空, 显 一个 灰色 占位 "跟 {AI名} 聊点什么试试 —— 文本 · 上传 PDF/图片/Word · 麦克风 语音输入. 内容 不会 保留."
- (e) 右侧 有 一个 "🃏 AI 底牌" sidebar (lg+ 屏宽 默认 显), 含 人格 / 语气 / 边界 / 触发关键词 / 绑定KB 列表
- (f) 底部 有 输入栏: 📎 上传按钮 + 🎤 麦克风按钮 + textarea + "发送" 按钮

**Fail**:
- 还显 "🚧 敬请期待" → mock 没替换
- 没有 调试模式 amber banner → 顶部 chrome 没渲

## T-02 · 纯文本 chat 走通 (SSE 流式) ★

**前置**: T-01 pass.

**步骤**:
1. 输入框 输 "你好, 请用一句话 介绍 你自己"
2. 按 Enter (或 点 发送)
3. 观察 主对话区

**Pass**:
- (a) 自己 的 消息 立刻 显示 在 右侧 (紫色 气泡)
- (b) 几秒 内 出现 AI 的 气泡 (左侧, AI 颜色), **字 是 一个一个 流式 出现** (有 ▌ 闪烁 光标)
- (c) 流完 后 ▌ 消失, AI 气泡 完整 显示
- (d) 顶部 banner "今日剩余" 数字 -1
- (e) **后端 不应** 写 任何 表 (没 meeting_transcript / 没 meeting_agent_message — 可在 SSH 查 DB 验证, 但 不强制)
- (f) DevTools → Network → 看到 POST `/api/agents/<id>/chat`, content-type `text/event-stream`, 状态 200

**Fail**:
- AI 气泡 不出现 / 流不动 → SSE 没工作
- 数字 不变 → quota tracking 没生效

**复述**: 截图 主对话区; 贴 一行 实际 SSE 帧 (DevTools Network → Response).

## T-03 · 刷新 / 切页 后 history 行为 ★ (核心 调试 语义)

**前置**: T-02 pass, 主对话区 已有 ≥ 2 条 消息.

**步骤 A: 同 tab 刷新**:
1. F5 / Cmd-R 刷新
2. 看 主对话区

**Pass A**:
- 消息 **仍在** (sessionStorage 保住了)
- "今日剩余" 重置 拉 (因为 后端 拿 不到 旧值, 直到 下次 发消息)

**步骤 B: 新开 tab**:
1. 复制 当前 URL, 新 tab 打开
2. 看 新 tab 主对话区

**Pass B**:
- 新 tab 主对话区 **空** (sessionStorage 不跨 tab)
- 这 验证 "调试模式" 严格 隔离

**步骤 C: 关 浏览器 重开**:
1. 完全 关 浏览器
2. 重开 + 登录 + 进 `/chat/<id>`

**Pass C**:
- 消息 **不在了** (浏览器关 = session 清)
- 顶部 banner 还在 (UI 不变)

**Fail**: 跨 tab 还能看到对话 → sessionStorage 错配 用了 localStorage / cookie

## T-04 · 文件上传 解析 + AI 看懂 ★

**前置**: T-01 pass; 准备 一个 ≤ 20MB 的 PDF 或 Word 文件 (内容随意, 但 有 文字)

**步骤**:
1. 点 输入栏 📎 按钮 → 选 文件
2. 等 toast: `"✅ <文件名> 已解析 (N 字)"` (N > 0)
3. 输入栏 上方 出现 一个 紫色 chip 含 文件名 + 字数
4. 输入框 输 "总结 一下 我刚上传的 文档"
5. 发送

**Pass**:
- (a) toast 正常 弹 (文件 真的 解析了)
- (b) chip 真的 出现 (✕ 可以 移除)
- (c) AI 回复 中 **明确 提及** 文件 里的 内容 (不是 编的)
- (d) 后端 不存 文件 (SSH 看 OSS / 本地 文件系统 没 新文件 — 不强制 但 应该 验)
- (e) 单文件 > 20MB → toast 错误 "文件 太大" (可拿 一个 大文件 验证)

**Fail**:
- AI 回复 跟 文件 完全 无关 → 文件 没传给 LLM
- toast 没弹 → parse-file endpoint 挂

## T-05 · 语音输入 (FunASR) ★

**前置**: T-01 pass; 浏览器 允许 麦克风 权限.

**步骤**:
1. 点 输入栏 🎤 按钮 → 应该 变成 红色 ⏹ 图标 (脉动 动画)
2. 对着 麦克风 念 一段话 (例 "你能 帮我 分析 这周 销售数据 吗?")
3. 等 几秒, 输入框 应 自动 填上 转写文字
4. 点 ⏹ 停止 (变回 🎤)
5. 检查 输入框 内容

**Pass**:
- (a) 录音 期间 麦克风 按钮 红色 + pulse 动画
- (b) 输入框 出现 ASR 转写 文本 (跟 你 说的话 大致 一致, 允许 小错)
- (c) 多 一句 → 文字 追加 (空格 分隔)
- (d) 停止 后 输入框 内容 不丢, 可 手动 编辑 + 发送
- (e) Network 看到 WS `wss://.../ws/chat-stt` 连接 + 二进制 帧
- (f) 后端 不写 transcript 表 (不强制 SSH 查; 但 这是 设计意图)

**Fail**:
- 点 🎤 弹 麦克风 错误 → 浏览器 没授权 (可重试)
- 录音 一直不出文字 → WS 连不上 / FunASR 没 启

## T-06 · AI 底牌 sidebar 真显 + 调试信息 ★

**前置**: T-01 pass; 选 一个 有 nickname / domain / persona / keywords / 绑 KB 的 agent.

**步骤**:
1. 看 右侧 sidebar
2. 发 一条 消息, 等 AI 回完
3. 再 看 sidebar 底部

**Pass**:
- (a) Sidebar 上方 显 agent 的 persona / 语气 / 边界 / 触发关键词 / 绑定 KB 列表
- (b) KB 列表 每项 是 一个 可点 链接 (跳 /me/profile/knowledge/<kb_id>)
- (c) AI 回完 后 sidebar **底部** 出现 一个 紫色 卡片 "📊 上一次 召回", 含 `KB chunks: N` 和 `memory: M` 两个 数
- (d) 如果 该 agent 绑了 KB, N 应 > 0 (KB 真被 读了)

**Fail**:
- Sidebar 完全 不显 → 折叠没打开 (点右上角 "底牌" 切换)
- 调试 卡片 不出现 → chat_debug_info SSE 帧 没接

## T-07 · per-user 日配额 卡 50 次 ★ (反滥用)

**前置**: T-02 pass. 这测 **不要 真跑 50 次** — 用 curl 模拟.

**步骤**:
1. 拿 owner cookie
2. curl 模拟 50 次 chat:
   ```bash
   for i in $(seq 1 50); do
     curl -sS -o /dev/null -w "%{http_code}\n" -b "<cookie>" \
       -H "Content-Type: application/json" \
       -d '{"messages":[{"role":"user","content":"hi"}]}' \
       https://aimeeting.zhzjpt.cn/api/agents/<active-agent-id>/chat
   done | sort | uniq -c
   ```
3. 第 51 次:
   ```bash
   curl -sS -w "\nHTTP %{http_code}\n" -b "<cookie>" \
     -H "Content-Type: application/json" \
     -d '{"messages":[{"role":"user","content":"hi"}]}' \
     https://aimeeting.zhzjpt.cn/api/agents/<id>/chat
   ```

**Pass**:
- 前 50 次 全 200
- 第 51 次 **429**, body 含 `"调试模式 日配额 已 用完"`

**Fail**:
- 51 次 还 200 → 配额 没生效

**注**: 配额 是 in-memory 计数, 重启 backend 会 重置. 如果 同时 多人 测 互不影响 (每个 user 独立).

## T-09 · 进入 私聊页 → invoke_count 真 +1 (调试 也 算 调用)

**前置**: T-02 pass. 拿 当前 agent 的 invoke_count (称 N0).

**步骤**:
1. curl GET `/api/agents/<id>` → 看 `invoke_count` (= N0)
2. 在 私聊页 发 1 条 消息, 等 AI 回完
3. curl GET `/api/agents/<id>` → 看 新值 (= N1)

**Pass**:
- N1 = N0 + 1 (严格 +1)
- 回 首页, 该 agent 卡片 "💬 N 次使用" 数字 = N1

**Fail**:
- N1 = N0 → 调试模式 invoke_count 没 +1, 跟 设计 不符 (调试 也 应该 算 使用)
