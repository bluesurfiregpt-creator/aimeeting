## 3 · 用例 T-01 ~ T-04

### T-01 · 撤销 endpoint 工作 (leader)

**前置**: P-1 + P-2 pass, 记下 superseded message id `<MSG_ID>` 和 `<MEETING_ID>`

**步骤**:
```bash
curl -sw "\n[HTTP=%{http_code}]\n" -b /tmp/cookies.txt -X POST \
  https://aimeeting.zhzjpt.cn/api/meetings/<MEETING_ID>/agent-messages/<MSG_ID>/restore
```

**Pass 条件**:
- HTTP 200
- response JSON: `{id, status: "active", superseded_by_message_id: null, restored: true}`
- 再 查 DB:
```sql
SELECT id, status, superseded_by_message_id FROM meeting_agent_message WHERE id=<MSG_ID>;
```
   应 看到 `status: active`, `superseded_by_message_id: NULL`

**Fail 条件**:
- HTTP 403/500
- response `restored: false` (但 DB 仍 superseded)
- DB 字段 没 改

**必须复述**: HTTP code + body + DB SQL 输出 全部

### T-02 · 撤销 idempotent

**前置**: T-01 pass (该 message 已 active)

**步骤**: 同 URL 再 POST 一次

**Pass 条件**:

- DB 仍 active

**Fail 条件**:
- HTTP 400/500
- DB 状态 改了

### T-04 · DOM drawer 可点 + 显 chain

**前置**: T-01 pass, P-1 找 一条 仍 superseded 的 message

**步骤**:

- 撤销按钮 显示 (leader 登录) — text "撤销 覆盖"

**Fail 条件**:

- leader 看不到 撤销按钮 / member 看到

**必须复述**: drawer DOM 结构 + chain 卡数量 + 按钮 文字 字面
