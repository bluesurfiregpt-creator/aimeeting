1. 只看真实页面 / 真实 HTTP 响应.

## T-01 · 任务列表卡 → 进详情 (待你处理 tab, source_kind=action)

### 前置 P-* PASS.

### 步骤
1. 进 /m/tasks, 默认 "等你处理" tab.
2. 找一张 `source_kind=action` 的卡 (有"确认/驳回"两个按钮 + 顶部 source_meeting 标签).
3. 看卡内 header 下方应有一行 **"查看任务详情 →"** (蓝色 link).
4. 点该 link.
5. 观察:
   - URL 变成 `/m/tasks/<action_item_id>`.
   - 进入新页, 顶部 sticky TopBar 含 ← 返回 + "任务详情" 标题.

### Pass (全部)
- 在 confirm 卡上看到 "查看任务详情 →" 链接.
- 点击后 URL 改变, 新页面渲染.
- 新页面 URL 含 action_item_id.
- 在 draft 卡 (`source_kind=draft`) 上**不应该**显示这个链接.

### Fail
- 链接没出现.
- draft 卡也显示了 "查看任务详情" (说明逻辑错).
- 点了 alert 弹出 (说明没上 P4.3).

### 必须复述
- 列表中 source_kind=action 卡截图 (含"查看任务详情 →").
- 列表中 source_kind=draft 卡截图 (无该链接).
- 点击前后 URL.

## T-02 · 详情页 Header — 任务全文 + 状态 chip + 截止 + 来源

### 前置 T-01 PASS.

### 步骤
1. 在详情页, 看 main 区域第一个 section (Header).
2. devtools 抓 `GET /api/m/tasks/<id>` JSON.

### Pass (全部)

  - 若有归属: AI 主责 (🤖 violet) 或真人主责 (👤 emerald) 或 hint (灰).
- 末行若有 source_meeting_id + title: 蓝色 link "来自 <meeting_title> →" 点跳 `/m/meetings/<id>`.

### Fail
- 任务内容截断显示.
- 任一 chip 跟 JSON 字段不一致.

### 必须复述
- 截图 Header 整段.
- JSON 的 content/status/due_at/is_overdue/assignee_* 字段值.

## T-05 · 评论区 — 列表 + 发送

6. 观察:
   - 按钮变 "发送中…" disabled.
   - 几百 ms 后弹绿色 toast "评论已发布".
   - 新评论出现在列表底部 (时间正序).
   - textarea 清空.
   - 字数计数回 "0/2000".

### Pass (全部)
- toast 绿色 "评论已发布".
- 新评论作者名 = 当前 user 名字 (owner 是 bluesurfiregpt).
- 评论 content = "P4.3 测试评论".
- 评论时间显示 "今天 HH:MM".
- 该评论右上角有 "删除" 链接 (因为是本人发的, can_delete=true).
