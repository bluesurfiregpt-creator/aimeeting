### T-08 暂停 → 恢复 → 继续走到终态

**前置**:T-07 pass
**步骤**:
1. 在 phase=running 时点 `暂停` 按钮
2. 立刻 截图 phase 徽章
3. 等 5 秒(确认 AI 不再讲新话,即消息区行数稳定)
4. 点 `恢复` 按钮
5. 等到 phase 变为 `done` 或 `已结束`(预计需要 1~3 分钟,看 LLM 速度)
6. 看页面顶部是否有 `分歧 banner`(紫色,文字含 `分歧待裁决`),记下有 / 无

**Pass 条件**:
- 暂停后 5 秒内 phase 徽章 字面变为 `已暂停` 或 `paused`
- 期间消息数 不再增加(±0,可能有 1 条 in-flight 落地这算正常,允许 +1 内)
- 点恢复后 phase 再次变 `运行中` / `running`
- 最终 phase 变为 `已结束` / `done`,颜色 emerald
- API 验证:
  ```bash
  curl -sS https://aimeeting.zhzjpt.cn/api/meetings/<M5>/consensus -b "<cookies>"
  ```
  - 返回数组长度 字面等于 `2`(因为 2 议程,每议程 1 行)
  - 每个元素的 `consensus_md` 字段 非空字符串
  - `dissents` 字段是数组(可能空可能非空,都 OK,只要不是 null)

**Fail 条件**:
- 暂停按钮 click 后 phase 没变 → fail
- 暂停后 5 秒内消息还在加 → fail("暂停信号未生效或 orchestrator 没读 phase")
- 终态不是 `done` 而是 `failed` → fail("会议失败,粘贴 last_error 字段")
- consensus 数组长度 ≠ 议程数 → fail("consensus 收集漏议程")

**必须复述**:
- 暂停前 / 暂停后 / 恢复后 / 终态 共 **4 次 phase 徽章截图**
- `GET /api/meetings/<M5>/consensus` 完整 JSON
- 分歧 banner 是否出现(写 "有,文字是 ...原文..." 或 "无,顶部只有 phase 徽章")
