## 测试用例

### T-01 · manager 进 /admin/agents 编辑自己的 agent → 看到 primary_user 名字 ★核心

**前置**: P-1, P-2 PASS, MGR1 已登录

**目标**: 验证 Kimi Issue 3 已修复 — manager 编辑自己 primary 的 agent 时,「绑定科室账号」字段应显示当前绑定 manager 的名字,而不是 "— 未绑 —".

**步骤**:
1. 打开 `https://aimeeting.zhzjpt.cn/admin/agents`
2. 在已有 Agent 列表里找到 AGENT_1 (primary = MGR1, 名字应该是 v26.5 那个 agent)
3. 点击 AGENT_1 行最右侧的 「编辑」 按钮 (✏️)
4. 滚动到左侧编辑表单的「🔗 绑定科室账号 (primary user)」section
5. 看该 section 实际显示内容

**Pass 条件** (必须全部满足):
- 看到 section 标题 `🔗 绑定科室账号 (primary user)`
- 看到一个 **不是 select 下拉框** 的纯文字框 (rounded box, 灰色背景)
- 文字框内容包含 **「当前绑定:」** + MGR1 的中文/英文名字 (绿色加粗)
- **不允许** 文字框内容是 `— 未绑 (本 AI 不能接任务) —`
- 框下方有提示文字 `🔒 转移 AI 给别的同事 需要 owner / admin / leader 权限`
- hover 文字框: tooltip 显示 `转移 AI 给别的同事 需要 owner / admin / leader 权限`

**Fail 条件**:
- ❌ 仍然显示 disabled select 下拉框
- ❌ 显示 `— 未绑 —` (说明 fix 失败)
- ❌ 显示空白 / loading

**必须复述**:
- 用 F12 看该 `<div>` 的实际文本内容
- 截图 整个编辑表单 (含「🔗 绑定科室账号」section)
- 复述 框内的 完整文字 (含 manager 名字 原文)
