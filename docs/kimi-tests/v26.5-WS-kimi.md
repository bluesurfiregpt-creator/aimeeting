## 测试用例

### T-01 · 顶栏 极简化 (4 项)

**步骤**: owner 登录后 看 顶栏右上角

**Pass**:
- 仅 4 个元素: ⚡超管(超管邮箱) + 🔔通知 + 「workspace名 · 👤 用户名」(整块可点) + "退出"
- ❌ 不应该看到: ⚙️ 后台齿轮 / 📊 数据看板 / 📨 上报问题 / ✏️ 下达指令 / 📬 消息中心

### T-02 · 顶栏 用户名 → 跳 /me/profile (工作站)

**步骤**: 点用户名

**Pass**:
- URL 跳 `/me/profile`
- 看到 **左侧 sidebar** 有多个分组 (🪪 我 / 🤖 AI 专家 / 📚 数据 / 🏢 工作空间 / ⚙️ 系统配置)
- 当前页 = 身份信息 (大头像 + 角色徽章)

### T-03 · sidebar 分组按角色显示

**步骤**: 4 个角色分别登录看 sidebar

**Pass**:
| 分组 | owner/admin/leader | manager | member |
|------|:------------------:|:-------:|:------:|
| 🪪 我 | ✅ | ✅ | ✅ |
| 🤖 AI 专家 (列表 + 模板生成器) | ✅ | ✅ (仅列表, 模板生成器 隐藏) | ✅ |
| 📚 数据 (KB/记忆/审批) | ✅ | ✅ | ✅ |
| 🏢 工作空间 (空间/成员/访问) | ✅ | ❌ | ❌ |
| ⚙️ 系统配置 (LLM/ASR/声纹/巡检/日志) | ✅ | ❌ 仅声纹可见 | ❌ 仅声纹可见 |

### T-04 · 老 /admin/* 自动跳转

**步骤**: 浏览器手输 `/admin/agents` 回车

**Pass**:
- URL 立刻跳 `/me/profile/agents`
- 显示 AI 专家列表 (跟以前一样的内容,但被 sidebar 包围)
- 同样测: `/admin/knowledge`, `/admin/team`, `/admin/models`, `/admin/audit` 都 跳 `/me/profile/*`

### T-05 · /enroll 跳 /me/profile/voiceprints

**步骤**: 手输 `/enroll`

**Pass**: 跳 `/me/profile/voiceprints`, 显示 声纹录入页

### T-06 · 首页 / 干净化

**步骤**: 进 `/`

**Pass**:
- 顶部只 2 个链接: 📚 会议历史 · 👤 我的工作站
- ❌ 不再有: 录入声纹 / AI 专家配置 / LLM 模型 / 长期记忆 (这些都进 sidebar 了)
- 新建会议 表单 还在 (核心入口保留)

### T-07 · 工作站 sidebar 跳转无刷新 (next.js client navigation)

**步骤**: 在 /me/profile 点击 sidebar 各项

**Pass**:
- URL 切到 /me/profile/agents → 主区切到 AI 专家列表
- URL 切到 /me/profile/knowledge → KB 列表
- 切的时候 顶栏 + sidebar 不重渲染 (next.js 行为)
- 当前项有 蓝色 active 高亮

### T-08 · 沉淀审批 独立 tab

**前置**: 有 1 个 pending draft (可手 SQL 插或 真跑 task done 流程)

**步骤**: manager 登录 → 点 sidebar 「待我审批」

**Pass**:
- sidebar 该项 右侧 显示红 badge 数字
- 进入 `/me/profile/sedimentation` → 3 个 tab (待我审批 / 已批准 / 已驳回)
- 第一个 tab 显示 pending 列表

### T-09 · AI 模板生成器 預留页

**步骤**: owner 进 sidebar 「AI 模板生成器」

**Pass**:
- URL = `/me/profile/agents/template`
- 看到 "🚧 即将上线 · v26.6 计划" + 功能轮廓
- 右下 有「→ AI 专家列表」按钮

### T-10 · 工作空间设置页

**步骤**: owner 进 sidebar 「空间设置」

**Pass**:
- URL = `/me/profile/workspace`
- 显示 workspace 名 / slug / id (只读)
- 「演示数据」section 有跳 `/me/profile/demo-data` 链接
