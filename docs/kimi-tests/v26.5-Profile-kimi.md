## 测试用例 T-01 ~ T-08

### T-01 · 顶栏点击姓名 → 跳 /me/profile ★

**前置**: P-2 PASS

**步骤**:
1. 在 任意页面 (例: `/`),点击 顶栏右上角的 **MGR1 的名字** (不是 ⚙️/📊/退出)
2. 看 URL + 页面内容

**Pass 条件**:
- URL 跳到 `/me/profile`
- 页面标题显示 `👤 个人中心`
- 右上角有 `→ 我的任务中心` 链接

**Fail 条件**: 跳到 `/me` (旧行为) / 跳到 `/admin/agents` / 不跳

**必须复述**:
- 截图 URL 栏 + 页面顶部
- F12 看 `[data-testid="me-profile-link"]` 的 href 属性

---

### T-02 · 个人中心 身份卡片 — 角色 + 工作空间 + 科室 ★

**前置**: T-01 PASS, 已在 `/me/profile`

**步骤**:
1. 看页面左列 第一张卡片 (身份卡片)
2. 看头像 / 名字 / 邮箱 / 角色徽章 / 工作空间 / 科室

**Pass 条件**:
- 大头像 (2 字母 initials, 紫色渐变背景)
- 名字: `Kimi 房屋安全科长` (或 MGR1 实际名字)
- 邮箱: MGR1 email 原文
- 角色徽章: **紫色** `部门 AI 维护人`
- 📍 工作空间: 显示 workspace 名 (例: `Kimi 综合测试 ws`)
- 🏢 科室: 显示 `房屋安全` (从 AGENT_1.domain 拼接而来) — **关键: 不是空**

**Fail 条件**:
- 邮箱显示 "(无邮箱)"
- 角色徽章是 红色/灰色 (应该紫色)
- 🏢 科室 不显示 / 显示 "-"

**必须复述**:
- 截图整张卡片
- 复述徽章颜色 (紫/红/灰) + 文字原文
- 复述 🏢 后的科室字符串原文

### T-03 · 个人中心 我维护的 AI 列表 + 跳转 ★

**前置**: T-02 PASS

**步骤**:
1. 看左列 第二个 section: `🤖 我维护的 AI 专家`
2. 数清楚有几个 AI
3. 点其中一个 AI 行右侧的 `✏️ 编辑` 链接

**Pass 条件**:
- Section 标题: `🤖 我维护的 AI 专家 (N)` — N 是数字 (MGR1 应该是 1, 那个 AGENT_1)
- 列表每行: 颜色 dot + AI 名字 + 领域 (例: 领域: 房屋安全)
- 右上角有 `→ 管理 AI` 链接 跳 `/admin/agents`
- 点 `✏️ 编辑` 跳 `/admin/agents`
- 跳过去后 看到 ⭐ 标记 在 AGENT_1 行

**Fail 条件**:
- 列表为空 (说明 backend primary_agents 没返)
- 跳错地方
- AGENT_1 行没 ⭐

**必须复述**:
- 截图整个列表
- 复述 AI 名字原文
- 跳 `/admin/agents` 后 截图 ⭐ 那一行

### T-04 · 个人中心 任务速览 + 跳任务中心

**前置**: T-02 PASS

**步骤**:
1. 看左列 第三个 section: `📋 我的任务速览`
2. 看 待签收 / 办理中 / 待审核 3 个数字
3. 点 `→ 任务中心` 链接

**Pass 条件**:
- 3 个数字 (没任务时 显示 `✨ 没有待处理的任务`,也算 PASS)
- 数字加底色: 待签收=amber / 办理中=sky / 待审核=violet
- 链接跳 `/me`

**Fail 条件**:
- 数字是 NaN / undefined / 异常字符
- 跳错地方

**必须复述**: 截图 + 数字原文

---

### T-05 · 个人中心 改名 ★

**前置**: T-02 PASS

**步骤**:
1. 看右列 `⚙️ 账户设置`
2. 点 `✏️ 修改名字` 按钮
3. 在弹出 dialog 中,把名字改成 `Kimi-profile-test-${当前时间戳}`
4. 点 `保存`
5. 看 toast + 页面

**Pass 条件**:
- toast 绿色: `✅ 名字已更新`
- dialog 关闭
- 身份卡片中的名字 已更新 (变成新名字)
- 头像 initials 也 变了
- PATCH `/api/auth/me` 返回 200

**Fail 条件**:
- 报 400 / 403 / 422
- 名字没改
- 页面没刷新

**必须复述**:
- 截图新名字
- 复述 PATCH 返回 body

### T-06 · 个人中心 改密码 (失败 — 旧密码错) ★

**前置**: T-02 PASS

**步骤**:
1. 点 `🔒 修改密码`
2. 旧密码 故意填错: `wrongpwd123`
3. 新密码: `newpassword2026`
4. 再次输入: `newpassword2026`
5. 点 `保存`

**Pass 条件**:
- toast 黄色: `权限不足 旧密码错误`
- dialog 不关闭
- 后端返回 403 `[权限不足] 旧密码错误`

**Fail 条件**: 误放行 / 报 500

**必须复述**: 截图 toast + body

### T-07 · /me 任务中心 顶部 身份摘要条 ★

**前置**: P-2 PASS, MGR1 登录

**步骤**:
1. 访问 `/me`
2. 看页面顶部 (H1 "我的工作台" 下面)

**Pass 条件**:

- 紫色小 dot (左侧)
- 🤖 维护 N 个 AI 这部分 是紫色 link, hover 显示 "查看我维护的 AI 列表"
- 点 🤖 维护 N 个 AI → 跳 `/me/profile`
- 右上角有 `👤 个人中心 →` 链接 跳 `/me/profile`

**Fail 条件**:
- 完全没看到这条
- 维护 0 个 AI (但 backend 明明有 1 个)
- 跳转失败

**必须复述**: 截图 + 复述这一行 完整文字

### T-08 · /admin/agents 联动 (manager 视角 ⭐ + 跳个人中心) ★

**前置**: P-2 PASS, MGR1 登录

**步骤**:
1. 访问 `/admin/agents`
2. 看顶部 紫色 "👋 部门 AI 维护人视角" banner
3. 看 banner 右侧 是否有 `👤 我的身份 →` 按钮
4. 看列表中 AGENT_1 (MGR1 primary 的那个) 是否有 ⭐ 标记
5. 点 `👤 我的身份 →`

**Pass 条件**:
- banner 右上角 有 紫色 `👤 我的身份 →` 按钮 (`data-testid="agents-to-profile-link"`)
- AGENT_1 行 名字旁边 有 amber `⭐ 我维护` 徽章
- 别的 agent (例: AGENT_2) **没有** ⭐ 徽章
- 点按钮跳 `/me/profile`

**Fail 条件**:
- ⭐ 出现在 别人的 AI 上 (说明逻辑错)
- 跳错地方

**必须复述**: 截图 banner + 列表 ⭐ 行
