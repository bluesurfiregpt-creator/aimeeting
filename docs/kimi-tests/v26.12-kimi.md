## T-03 · 后端 list_agents 新 query params 工作 ★

**前置**: T-01 pass, owner cookie 已拿到.

**步骤** (4 个 curl):

```bash
# 3a · 默认 (无参数)
curl -sS -b "<cookie>" https://aimeeting.zhzjpt.cn/api/agents | head -1
# 3b · 搜索
curl -sS -b "<cookie>" "https://aimeeting.zhzjpt.cn/api/agents?q=房屋"
# 3c · 排序 hot
curl -sS -b "<cookie>" "https://aimeeting.zhzjpt.cn/api/agents?sort=hot"
# 3d · domain 筛选
curl -sS -b "<cookie>" "https://aimeeting.zhzjpt.cn/api/agents?domain=楼宇安全&active_only=true"
```

**Pass**:
- (3a) 返回 JSON 数组, 每个 元素 含 `invoke_count` (number) + `nickname` (string|null) 两个 字段
- (3b) 仅 含 name/nickname/persona/domain 中 任一 包含 "房屋" 的 agent (其他 排除)
- (3c) 第一个 元素 的 invoke_count >= 第二个元素 的 invoke_count (降序)
- (3d) 仅 含 domain="楼宇安全" 且 is_active=true 的 agent

**Fail**:
- 字段 缺 `invoke_count` / `nickname` → schema 没 同步
- 搜索 返回 不相关 agent → ILIKE 没生效
- hot 排序 后 第一个 < 第二个 → ORDER BY 错

**复述**: 贴 (3a) 第一个 agent 的 完整 JSON 原文.

## T-04 · invoke_count 真的 +1 ★ (核心)

**前置**: T-03 (3a) pass — 拿 到 一个 agent 的 当前 `invoke_count` 值 (称作 N0).

**步骤**:
1. owner 新建 一场 hybrid 会议, 邀请 这个 agent (call 它 A)
2. 进 `/meeting/<id>`, 点 "▶ 开始会议", 等 ASR 连上
3. 在 会议室 AI 画廊 点 agent A 的 头像 (手动 invoke)
4. 等 5-10 秒, 看 agent A 的 流式回复 在 中栏 出现
5. 等回复 流式 结束 (流停 + 不再 加新 字)
6. curl GET `/api/agents/<A.id>` → 拿 当前 invoke_count (N1)

**Pass**:
- N1 = N0 + 1 (严格 +1, 不是 +2 不是 +0)

**Fail**:
- N1 = N0 → invoke_count 没 +1 (agent_router 改 没生效 / commit 失败)
- N1 = N0 + 2 → 重复 计数
- N1 > N0 + 1 → 多次 invoke 错累加

**复述**: N0 字面值 = ? N1 字面值 = ? 差值 = ?
