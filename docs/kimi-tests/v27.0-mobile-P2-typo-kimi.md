## T-01 · `/m` 默认页字号 — 所有可读内容 ≥ 13px

### 步骤
1. 在 `/m` 页 (会议视角默认), 用浏览器 devtools 抽样 5 个文本元素
2. 抽样: PageHeader title, SegmentControl 各 tab label, HeroOngoingCard 标题 + 状态行 + AI insight 内容, MiniListRow title + 副标题, AI 智囊议题聚合行 title + count chip

### Pass
- 所有抽样的可读文字 (排除红点 badge 数字) `computed font-size ≥ 13px`
- SegmentControl 各 tab label `font-size` 在 `14-15px` 范围
- HeroOngoingCard 状态行 (进行中 · X min) ≥ 14px

### Fail
- 任一抽样文字 `font-size < 13px`.

### 必须复述
- 每个抽样元素的 selector + computed font-size, 5 个全列.

## T-02 · 专家工卡 (重点修复) — 字大 + 布局变化

### 步骤
1. `/m` segment 切「专家视角」
2. 找到第一张工卡 (任意 active expert)
3. 看下半区结构 — 应该是「最近会议」section + 横线 + 「任务」section, 而不是 grid-cols-2 两个小方块

### Pass 条件 (全部满足)
- 工卡内不再有 grid-cols-2 子卡结构. 改成纵向 section.
- 会议条目里, 会议标题 `font-size ≥ 15px` (旧版 12px), 日期 `font-size ≥ 14px` (旧版 11px).
- 任务大数字 (3 进行中) 数字部分 `font-size ≥ 17px` 且 `font-weight ≥ 600`.
- 「最近会议」「任务」section 标题 `font-size ≥ 14px`.
- Header 上「未激活」chip 或时间戳 `font-size ≥ 13px` (旧版 11px).

### Fail
- 看到任意 `font-size < 13px` 在工卡可读内容内.
- 工卡仍然是 grid-cols-2 的两个 ink-800 子方块结构.

### 必须复述
- 截图整张工卡.
- 列出 5 个抽样元素的 selector + computed font-size.
