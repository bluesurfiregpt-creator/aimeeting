## 3 · 用例 T-01 ~ T-04

### T-01 · NLU 真接 (确认不 mock)

**前置**: P-2 pass

**目标**: 调 `/api/v2/mira/draft-meeting` 两次, 不同 input → 不同 output (mock 时 永远 一样).

**步骤**:
1. 第一次 调用:
```bash
curl -s -b /tmp/cookies.txt -X POST https://aimeeting.zhzjpt.cn/api/v2/mira/draft-meeting \
  -H "Content-Type: application/json" \
  -d '{"input_text":"评估 电梯改造 方案 法务 风险, 让 Lex 看看 业主 同意 流程","input_mode":"text"}'
```
2. 第二次 调用 不同 input:
```bash
curl -s -b /tmp/cookies.txt -X POST https://aimeeting.zhzjpt.cn/api/v2/mira/draft-meeting \
  -H "Content-Type: application/json" \
  -d '{"input_text":"Q3 路线图 对齐, 三个 epic 优先级, 数据 团队 + 架构 一起","input_mode":"text"}'
```

**Pass 条件**:
- 两次 `proposed_title` **不同** (真 LLM 看 input 拟标题)
- 两次 `proposed_agenda` 内容 **不同**
- 第一次 `proposed_ais` 含 `name: "Lex"` (法务 上下文)
- 第二次 `proposed_ais` 含 `name: "Stratos"` 或 `name: "Sage"` (架构/数据 上下文)
- `proposed_ai_names` 字段 全部 ⊆ 真实 agent 名单 (不出现 "ai-mira" 这种 mock 假 id)

**Fail 条件**:
- 两次 `proposed_title` 一字 不差 = "搜索改版上线前评审" (老 mock)
- agent id 以 "ai-" 开头 (老 mock 假 id 没替)

**必须复述**: 两次 完整 response JSON (含 proposed_title + proposed_ais 字面 数组)

### T-02 · LLM 失败 fallback 不挂

**前置**: T-01 pass

**目标**: 发 一个 LLM 必 拒绝 的 input → 验 fallback mock 返回 (不 500)

**步骤**:
```bash
curl -sw "\n[HTTP=%{http_code}]\n" -b /tmp/cookies.txt -X POST \
  https://aimeeting.zhzjpt.cn/api/v2/mira/draft-meeting \
  -H "Content-Type: application/json" \
  -d '{"input_text":"a","input_mode":"text"}'
```

**Pass 条件**:
- HTTP 200 (不 500)
- 返回 含 `proposed_title` / `proposed_agenda` 字段 (即使 LLM 给空 也 fallback 兜底)
- `confidence` ≤ 0.6 (说明 LLM 不确定 / 走 fallback)

**Fail 条件**:
- HTTP 500 (代码 没 try/except)
- 返回 空 body 或 null

**必须复述**: HTTP code + response 完整 JSON

### T-04 · attendee 真 关联

**前置**: T-03 pass

**目标**: 新创 meeting 应 有 attendee 行 (agent + 真人)

```sql
SELECT m.title, ma.user_id, ma.agent_id, a.name AS agent_name, u.name AS user_name
FROM meeting m
LEFT JOIN meeting_attendee ma ON ma.meeting_id = m.id
LEFT JOIN agent a ON a.id = ma.agent_id
LEFT JOIN "user" u ON u.id = ma.user_id
WHERE m.id = '<T-03 创的 meeting id>'
ORDER BY ma.role;
```

**Pass 条件**:
- 至少 1 行 attendee, agent_name 或 user_name 非空
- agent_id 是 真 workspace agent (跟 LLM 返回 的 id 一致)

**Fail 条件**:
- 0 行 attendee — 创会 没 bind
- agent_id 是 mock id (`ai-lex` / `u1` 等) — 前端 没过滤 mock 残留

**必须复述**: SQL 输出 完整行
