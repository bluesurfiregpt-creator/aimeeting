## 顶部 6 条死规矩 (反幻觉)

1. 只看真实页面 / 真实 HTTP 响应.
2. 每步判定必须基于本步实际看到的数据.
3. 复述要原文 (带引号的精确字符串).

## T-01 · 工卡点击 → 跳详情页 (不再 alert)

### 前置 P-* PASS.

### 步骤
1. /m segment「专家视角」, 看到工卡.
2. 点击任一工卡 (整张卡可点击区域).
3. 观察 URL 和页面变化.

### Pass
- URL 变成 `https://aimeeting.zhzjpt.cn/m/agents/<uuid>` (uuid 跟该专家 agent_id 一致).
- 页面切到新页, 顶部能看到该专家名 + 返回 ← 按钮.
- 不出现 alert 弹窗.

### Fail
- 弹出 alert "Phase 3: 跳 /m/agents/...".
- URL 没变化, 仍在 /m.

### 必须复述
- 点击前 URL + 点击后 URL (复制粘贴整串).
- 截图详情页顶部.

## T-02 · 详情页档案区显示累计统计

### 前置 T-01 PASS.

### 步骤
1. 在详情页, 看 TopBar 下方第一段 "档案区" 卡.
2. 同时手动调 `GET /api/m/agents/<id>` 看 JSON.

### Pass (全部)
- 档案卡显示 3 个数字: `N 场会议` + `M 条智囊` + `K 项任务`.
- 数字字号 ≥ 18px, font-weight ≥ 600.
- 数字跟 JSON 字段一一对应:
  - 场会议 = `total_meetings`
  - 条智囊 = `total_insights`
  - 项任务 = `tasks.length`
- 下方 caption "最近活跃 X 时间前" 或 "暂未激活".

### Fail
- 任一数字跟 JSON 不一致.
- 数字字号 < 18px.
- 字段缺失.

### 必须复述
- 截图档案卡.
- 完整粘贴 JSON 的 `agent_id` / `name` / `total_meetings` / `total_insights` / `last_active` 五个字段值.

## T-03 · Segment 切换 — 会议 / 任务 / 智囊 三 tab

### 前置 T-02 PASS.

### 步骤
1. 详情页 segment 区域应该有三个 tab: "会议" + "任务" + "智囊", 每个后面跟 (N) 数字.
2. 默认 tab 是 "会议".
3. 点击 "任务" → 主区域内容切换. 点 "智囊" → 再切.

### Pass (全部)
- 三个 tab label 完全是 "会议" / "任务" / "智囊" 这三字串.
- 每个 tab 后面有数字, 跟 JSON `meetings.length` / `tasks.length` / `insights.length` 一致.
- 切 tab 时主区域明显变 (前后不同 DOM 子树).

### Fail
- 任一 tab label 不一致.
- 切换无反应.

### 必须复述
- 截图 segment 状态 1 (默认会议).
- 截图 segment 状态 2 (切到任务).
- JSON 三个数组长度.

## T-04 · 会议 tab — 列表 + 每条显示 insight 计数

### 前置 T-03 PASS.

### 步骤
1. 在 "会议" tab, 看列表项.
2. 任一会议卡含: 状态 chip + 日期 / 时间前缀 + 标题 + (可选) "💡 该专家在此会产出 N 条智囊" 注释.

### Pass
- 至少一条会议卡能完全渲染.
- 如该专家在某会议 `insights_count > 0`, 卡下方紫色文字 "💡 该专家在此会产出 N 条智囊", N 跟 JSON `meetings[i].insights_count` 一致.
- 如 insights_count == 0, 不显示该行 (OK 不算 fail).
- 点击会议卡跳 `/m/meetings/<meeting_id>`.

### Fail
- insights_count > 0 但 UI 不显示 💡 那行.
- 数字跟 JSON 不一致.

### 必须复述
- 任一会议 row 截图.
- 该 row 对应的 JSON `meetings[i]` 完整粘贴.

## T-05 · 任务 tab — 按状态分组 (进行中 / 已完成 / 已取消)

### 前置 T-03 PASS.

### 步骤
1. 切到 "任务" tab.
2. 看是否按 状态 分了 section. 「进行中」标题 + 列表, 「已完成」标题 + 列表, 「已取消」标题 + 列表.
3. 若任一 group 为空, 该 group 不显示 (OK).
4. 若 `is_overdue == true`, 卡有红色边框 + "超期" 红 chip.

### Pass
- "进行中" 数 = `tasks.filter(t => OPEN_STATES.includes(t.status)).length`,
  OPEN_STATES = {open, dispatched, accepted, in_progress, submitted}.
- "已完成" 数 = `tasks.filter(t => ['done','archived'].includes(t.status)).length`.
- 任务标题字号 ≥ 15px.
- 超期 chip 红色, 实际看到 "超期" 二字.

### Fail
- 分组逻辑错: 比如 in_progress 被算到「已完成」.
- 任务列表完全空但 JSON tasks 有内容.

### Skip 允许
- 若该 agent 没分配任务 (`tasks.length == 0`), 显示空态 "该专家还没分配任务", 不算 fail.

### 必须复述
- 任务 tab 截图.
- JSON `tasks` 数组里每条的 `task_id` + `status` + `is_overdue` (列表式).

## T-06 · 智囊 tab — AIInsightCard 列表

### 前置 T-03 PASS.

### 步骤
1. 切 "智囊" tab.
2. 看每张卡: header (agent + type chip + 时间) + content + (可选) evidence + footer (来自 X 会议).

### Pass
- 卡用 AIInsightCard 渲, 内容字号 ≥ 15px.
- 卡片数量 = `insights.length` (上限 50).
- 任一卡 type chip 颜色跟 type 对应 (建议=violet / 风险=rose / 洞察=sky / 思路=amber / 决策建议=emerald).

### Fail

- 类型 chip 颜色全部相同 (没按 type 区分).

### Skip
- 该 agent 没产出 insight, 显示空态 "该专家暂无智囊产出", OK.

### 必须复述
- 智囊 tab 截图.
- `insights[0]` 完整 JSON.
