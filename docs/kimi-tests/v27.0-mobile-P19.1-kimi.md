## T-01 · 专家卡 默认 折叠 — 仅显头部

### 前置 P-* PASS.

### 目标
首屏 渲染 时 所有 卡片 都 折叠 (无 "最近会议" / "任务" sections),
但 数字摘要 出现 在 头部 下方.

### 步骤
1. 在 `/m` 专家视角 tab, 截图.
2. 滚动 看 前 5 个 卡片.

### Pass
- 每个 卡片 高度 ≈ 80-120px (远小于 之前 200-300px).
- 卡片内容 **不含** 字面 `📅 最近会议` 或 `📋 任务` (因 折叠).
- 卡片右上有 一个 圆形 灰底 按钮 含 字符 `▼` (不旋转, 折叠态).
- 头部 下方 有 一行 灰字 摘要, 格式形如:
  - `3 场会议` / `5 场会议 · 1 进行中` / `2 进行中 · 1 超期` 等
  - 用 ` · ` 分隔 多个 bit
- 没 meetings 且 没 tasks 的 卡片 → 摘要行 不显 (或为空).

### Fail
- 卡片 渲染 时 就 显 完整 内容 (含 "最近会议"/"任务" 字面) → 折叠未生效.
- ▼ 字符不在.
- 卡片 高度 > 200px (说明 折叠 没收) — 看截图.

### 必须复述
- 截图 (≥ 5 卡片 显视区).
- 复述 ≥ 2 张 卡片 的 "数字摘要" 行原文.

## T-04 · 上传 .pptx → 同步 抽取 + summary

### 前置 P-* PASS.

### 步骤
1. 进 `/m/meetings/new`.
2. 滚到 `参考资料` section, 点 `+ 添加文件` → 选 `fictional-Q1.pptx`.
3. devtools Network 看 POST `/api/meetings/attachments`.
4. 等 ≤ 10 秒.

### Pass
- response status `200`. body 含:
  - `filename: "fictional-Q1.pptx"`
  - `extension: "pptx"`
  - `extract_status: "ready"` (字面)
  - `extract_summary` 非空, 含 至少 1 个 PPT 内容词:
    `Q1` / `投诉` / `安保` / `542` / `Q2` / `50w` / `莲花山` 等.
- UI 卡片 显 `✓ 就绪` chip (emerald 色).

### Fail
- 400 `不支持的文件格式 .pptx` → 后端 白名单 漏加.
- 503 → OSS 没配.
- `extract_status: "failed"` → python-pptx 没装 / 出版本错.

### 必须复述
- response body 完整 JSON 原文.
- attachment 卡片 截图.

## T-06 · 上传 图片 → 走 OCR (extracting → ready)

### 前置 P-* PASS, DashScope OCR 已配.

### 目标
图片 不再 skipped — 走 Qwen-VL OCR, 抽出 文字 → ready 状态 + summary 含 文字.

### 步骤
1. 同 draft 上传 `sign-budget-50w.png` (含 文字 "预算 50w / Q2 整改").
2. response 立即 返回 — 看 `extract_status`.
3. 等 5-30 秒, 用 T-05 方式 GET draft list 再看 该附件 状态.

### Pass
- 步 2: response `extract_status: "extracting"` (字面 — 图片 一律 async).
- UI 立刻 显 紫色 `抽取中…` chip.
- 步 3: GET 后 该附件 `extract_status` 变 `ready` (或最差 `skipped` 若 OCR 没识别到字).
- ready 时 `extract_summary` 含 至少 1 个 图片文字 关键词: `预算` / `50w` / `Q2`.

### Skip
- DashScope 没配 → status 直接 `failed` (last_error 含 "OCR"). 标 SKIP, 不算 fail.

### Fail
- 步 2 立即 `skipped` + "OCR 未启用" 占位 → 老代码没被替换.
- ready 但 summary 完全 跟 图片 文字 无关 → OCR API 调通但 返回了别的内容,
  标 观察 + 给 OCR raw response.

### 必须复述
- 步 2 response body JSON 原文.
- 步 3 GET response 中 该 attachment 的 extract_status + summary 原文.
