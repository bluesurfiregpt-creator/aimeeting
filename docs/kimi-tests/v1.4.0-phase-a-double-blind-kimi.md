## 测试用例 T-01 ~ T-09

> **路径规范 (NORTH_STAR § 8.6)**: 以下 所有 命令 / 文件路径 都 用 `$REPO_ROOT/...` 前缀.
> 跑 命令 前 确认 `echo $REPO_ROOT` 输出 是 你 clone repo 的 绝对路径 (e.g. `/tmp/aimeeting`).

### T-01 · 剧本 A runner 顺利跑完

- **前置**: P-1 ~ P-3 pass + `$REPO_ROOT/scripts/blind-test-runner.py` 拿到
- **目标**: 验证 runner 端到端 跑通, 输出 result JSON
- **步骤**:
  ```bash
  TS=$(date +%Y%m%d-%H%M%S)
  env -u HTTP_PROXY -u HTTPS_PROXY -u http_proxy -u https_proxy \
    python3 $REPO_ROOT/scripts/blind-test-runner.py \
      --script $REPO_ROOT/docs/kimi-tests/blind-test/scripts/A-grayrelease.json \
      --email demo.lijg@futian.gov.cn \
      --password demo123 \
      --runner kimi \
      --out $REPO_ROOT/docs/kimi-tests/blind-test/results/run-kimi-A-${TS}.json
  ```
- **Pass 条件**:
  - 进程 0 退出 (exit code 0)
  - stdout 最后 1 行 是 JSON `{"meeting_id": "...", "agent_starts": N, ...}`
  - result JSON 文件 真生成 (`ls $REPO_ROOT/docs/kimi-tests/blind-test/results/run-kimi-A-*.json` ≥ 1 file)
- **Fail 条件**:
  - exit non-zero / 抛 exception
  - InvalidProxyMessage (proxy env var 没 unset)
  - "login failed" (账号 错)
- **必须复述**: stdout 完整 + 生成的 result 文件 完整绝对路径

### T-02 · 剧本 B runner 顺利跑完

- 同 T-01, 但 `--script $REPO_ROOT/docs/kimi-tests/blind-test/scripts/B-property.json`, `--out $REPO_ROOT/docs/kimi-tests/blind-test/results/run-kimi-B-${TS}.json`

### T-03 · 剧本 A 自动判定 item 2 (LLM judge 主动度)

- **前置**: T-01 pass + result JSON 在
- **目标**: 校验 AI 真主动 chime in (Phase A · 2 痛点 1).
- **步骤**:
  ```bash
  jq '.results.item_2_proactive_count, .results.item_2_avg_latency_s' \
    $REPO_ROOT/docs/kimi-tests/blind-test/results/run-kimi-A-*.json | tail -2
  ```
- **Pass 条件**: 
  - `item_2_proactive_count >= 2` (至少 2 个 AI 主动发言)
  - `item_2_avg_latency_s <= 8.0` (Phase A · 2 阈值)
- **Fail 条件**:
  - count = 0 → AI 没主动 (Phase A · 2 keywords / 阈值 失效)
  - latency > 8.0 → 阈值 没生效
- **必须复述**: 数字原文

### T-04 · 剧本 A 立场守门 (item 3)

- **前置**: T-01 pass
- **目标**: AI 真有立场, 不说和稀泥 (Phase A · 3 痛点 2).
- **步骤**:
  ```bash
  jq '.results.item_3_stance_violations, .results.item_3_stance_strong | length' \
    $REPO_ROOT/docs/kimi-tests/blind-test/results/run-kimi-A-*.json
  ```
- **Pass 条件**:
  - `stance_violations` array 为 空 (没 "都可以"/"看情况"/"两个都可以"/"各有利弊"/"都有道理")
  - `stance_strong` 命中 至少 1 个 "建议" 或 "因为" 词
- **Fail 条件**: violations 非空 (AI 和稀泥 触发)
- **必须复述**: stance_violations array 完整原文

### T-05 · 剧本 A summary v2 含 AI speakers (item 4)

- **前置**: T-01 pass
- **目标**: Phase A · 4 PM 拍 "修根" — AI 专家 进 summary, 不被剔除 (痛点 3).
- **步骤**:
  ```bash
  jq '{
    has_ai: .results.item_4_summary_has_ai_speakers,
    action_w_src: .results.item_4_action_items_with_source,
    topic_count: .results.item_4_topic_count,
    has_json: .results.item_4_summary_json_present,
  }' $REPO_ROOT/docs/kimi-tests/blind-test/results/run-kimi-A-*.json
  ```
- **Pass 条件**:
  - `has_ai: true` (核心: AI 进 summary 了)
  - `has_json: true` (结构化 summary 真生成了)
  - `topic_count >= 1`
  - `action_w_src >= 1` (任务溯源 chip 有数据)
- **Fail 条件**:
  - `has_ai: false` → Phase A · 4 没真 work (回归)
  - `has_json: false` → LLM JSON 输出 错位, fallback 走 markdown
- **必须复述**: jq 输出原文

### T-06 · 剧本 A AI 主持人 接力 (item 5) — **Round 2 metric 改 combined**

- **前置**: T-01 pass
- **目标**: AI 主持人 主动 推 接力 — 两种 形式 之一 (Phase A · 5 痛点 6):
  - `agent_recommendation`: orchestrator.recommend_next_speaker LLM judge 推荐 下一个 发言 agent
  - `dissent_detected`: dissent_detector LLM judge 检测 反对观点 + suggested_agent 接力 仲裁

  Round 1 (2026-05-27) PM 拍板 (路径 A): 改 metric 算 二者 之和 `item_5_combined_count`,
  因为 两类 事件 都是 "AI 主持人 接力" 不同 表达形式 (一个 是 主动 接力, 一个 是 仲裁 接力).
  老 metric `recommendation_count` 漏算 dissent → Round 1 Kimi 剧本 A 误判 fail (retro
  深审 后 确认 是 metric bug, 非 backend bug).
- **步骤**:
  ```bash
  jq '{
    recommend: .results.item_5_recommendation_count,
    dissent: .results.item_5_dissent_count,
    combined: .results.item_5_combined_count
  }' $REPO_ROOT/docs/kimi-tests/blind-test/results/run-kimi-A-*.json
  ```
- **Pass 条件**: `combined >= 1` (任一 形式 至少 1 次)
- **Fail 条件**: `combined = 0` → AI 主持人 完全没 主动 接力 (backend orchestrator + dissent_detector 都 没 fire)
- **必须复述**: 3 个 数字 原文 (recommend / dissent / combined)

### T-08 · 跟 Claude 结果 对账

- **前置**: T-03 ~ T-07 全过 + Claude 结果 JSON 已 commit (在 `$REPO_ROOT/docs/kimi-tests/blind-test/results/run-claude-*.json`)
- **目标**: 双 AI 结果一致 (PM Q7 拍 "Claude 与 Kimi 结果一致")
- **步骤**:
  ```bash
  for run in run-kimi-A-*.json run-claude-A-v4-*.json; do
    f="$REPO_ROOT/docs/kimi-tests/blind-test/results/$run"
    if [ -f "$f" ]; then
      echo "=== $run ==="
      jq '.results | {item_2_count: .item_2_proactive_count, item_2_lat: .item_2_avg_latency_s, item_3_violations: (.item_3_stance_violations | length), item_4_has_ai: .item_4_summary_has_ai_speakers, item_4_action_src: .item_4_action_items_with_source, item_5: .item_5_recommendation_count}' "$f"
  ```

  - Claude vs Kimi 的 item_2_proactive_count 差值 <= 2 (允许 LLM 概率 抖动)
  - Claude vs Kimi 的 item_4_summary_has_ai_speakers 必须 都为 true (binary 一致)
  - Claude vs Kimi 的 **item_5_combined_count 都 >= 1** (两 端 都 有 接力, recommend 或 dissent 任一)
  - item_3_stance_violations 都 = 0 (都 不和稀泥)
- **Fail 条件**:
  - Claude 7 项 PASS, Kimi 任一项 不到 阈值 — 双盲 不一致 (RED)
- **必须复述**: 完整两份 jq 输出 + 是否 一致 (PASS/FAIL)

## 双盲测试 真实价值 已 体现

**根因**: demo_seed_v2 给 agents seed 时 没 set keywords 字段 (`keywords=[]`).
agent_router.maybe_invoke_agents 检查 `if not ag.keywords: continue` → 直接 跳过.
**Phase A · 2 阈值 调优 (8s/30s → 4s/15s) 没解 这个 根** — 双盲 first 跑 就 暴露了.

**修复**: PATCH /api/agents/{id} 给 9 个 expert agents 加 keywords (2026-05-27 22:38) +
demo_seed_v2.py 加 keywords spec (commit 后续).

**Round 2 (2026-05-27 22:40)**: 同 剧本 A 跑 → 3 agents triggered (0.67s avg latency).

这是 PM 拍板 双盲测试 的 真价值 — **AI 测试 暴露 backend 配置 bug**, 不靠 人 真会议 试错.
