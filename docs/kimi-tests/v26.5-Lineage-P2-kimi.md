## 测试用例

### A. sidebar 重构

#### T-01 · sidebar 显示 🧬 AI 数据中心 大组 (合并)
**步骤**: owner 登录 → 点用户名 → 看 sidebar
**Pass**:
- 不再有 "🤖 AI 专家" + "📚 数据" 两个独立组
- 合并成一个 "🧬 AI 数据中心" 大组, 含 6 项:
  - 🌐 全景血缘图
  - 🤖 AI 专家
  - 📚 知识库
  - 🧠 长期记忆
  - 🔔 审批中心
  - ✨ AI 模板生成器 (仅 owner+)

### B. 桑基血缘图

#### T-02 · 全景血缘图渲染
**步骤**: 点 🌐 全景血缘图
**Pass**:
- URL = `/me/profile/lineage`
- 顶部 5 个 stat 卡片 (AI / KB 文档 / Memory / 来源 / 上传人) 显示数字
- 中间桑基图渲染出 3 列节点:
  - 左: 来源 (📁 上传人名 / 🎙️ 会议 / 📋 任务)
  - 中: 📄 KB 文档 / 🧠 Memory
  - 右: 🤖 AI 专家
- 边: 浅色彩色连线
- 图例 (5 种颜色)

#### T-03 · 单 AI 视角
**步骤**: 顶部 "视角" 下拉选 一个 AI
**Pass**:
- URL 不变, 数据重载
- 桑基图只显示 跟这个 AI 相关的节点
- stats 数字减少 (只统计本 AI 相关)

#### T-04 · 高亮 AI
**步骤**: 选 "高亮 AI = AGENT_1"
**Pass**:
- 只看到 涉及 AGENT_1 的链路 (其他被过滤掉)

#### T-05 · 共享数据过滤
**步骤**: 勾选 "只看共享数据 (≥2 AI 引用)"
**Pass**:
- 只显示 被 多个 agent 引用的 KB doc / Memory 节点
- 如果 workspace 全是 单 AI 独占 数据 → 显示 "没有数据"

#### T-06 · 节点点击详情
**步骤**: 点击某个 KB 文档 节点
**Pass**:
- 弹 modal 显示节点详情:
  - 类型 (📄 KB)
  - 名字
  - meta 字段 (kb_id / kb_name / source_type / chunk_count / data_classification 等)

### C. KB 反查 + Memory 多 select

#### T-07 · KB 列表 显示 被几个 AI 引用
**步骤**: 进 📚 知识库 页
**Pass**:
- 每个 KB 卡片底部 (📄 N 文档 · 🧩 N 分块 后面) 可能有 🔗 N AI 引用 徽章
- hover 显示 引用的 AI 名字列表

#### T-08 · Memory 多 select agent
**步骤**: 进 🧠 长期记忆 → 手工添加表单
**Pass**:
- 原 单 select 已替换成 多 checkbox 列表
- 第一个勾的显示 ⭐, 其余显示 🔗 N
- 添加成功 → 列表显示 1 条 memory 多个 agent 徽章 (⭐ + 🔗)

### D. 后端 API

#### T-09 · curl GET /api/lineage
```bash
curl -b cookie.txt https://aimeeting.zhzjpt.cn/api/lineage
```
**Pass**: 返回 {nodes, edges, stats}, nodes 含 5 种 type, edges 含 5 种 kind

#### T-10 · curl GET /api/lineage/agent/{id}
**Pass**: 仅 节点 跟该 agent 有关
