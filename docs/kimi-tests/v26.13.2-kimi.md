  ```bash
  # 不直接 暴露, 看 GET /api/me workspace 字段 — 或者 后端 SSH 看 DB
  ```

- (b) 没 新创建 草稿

## T-07 · 审批中心 见 perplexity_auto 草稿 + 批准 → 入 KB ★★

**前置**: T-05 pass — 已有 一个 pending perplexity_auto 草稿.

**步骤**:
1. 用 agent 的 primary_user 账号 (例 expert demo.fengl) 登录
2. 进 审批中心 (`/me/profile/审批中心` 或 类似路径)
3. 应 见 一个 待审 草稿, 含:
   - kind 标签 / 提议 文件名 (Perplexity · ...) / proposed_summary 是 markdown synth + 来源 URL 列表
4. 点 "批准"
5. 等 几秒 (chunk + embed)

**Pass**:
- (a) 草稿 status → "approved"
- (b) 该 KB 中 多 一篇 doc, filename 含 "Perplexity ·" 前缀
- (c) curl `/api/knowledge-bases/<kb_id>/documents` 看 该 doc 有:
  - `source_type = "perplexity_auto"`
  - `source_url` 非 null (Perplexity 第一个 citation URL)
  - `source_query` = 当时 输 的 query
  - `source_fetched_at` 非 null
- (d) doc.status = "ready", chunks 已 写

**Fail**:
- 批准 后 KB 没 多 doc → consolidator 失败
- doc 缺 溯源字段 → consolidator 没 写 meta

## T-08 · 私聊 闭环: AI 答不出 → "📚 用 Perplexity 补充" 按钮 ★

**前置**: T-02 pass; agent (例 公共住房建设AI专家) 绑了 KB; KB 中 没有 跟 测试 query 相关的内容.

**步骤**:
1. 进 该 agent 的 私聊页 `/chat/<agent_id>`
2. 输 一个 该 agent 答 不出 的 problem (例 "你 知道 2050 年 的 火星 殖民 政策 吗?")
3. 等 AI 回完

**Pass**:
- (a) AI 给 出 回复 (内容 不强求, 可能 是 "我不知道" / "目前没看到资料")
- (b) AI 气泡 **下方** 出现 紫色 卡片 含:
  - `"💡 本次 回复 没 引用 KB 中 任何 文档..."` (字面 reason)
  - 一个 "📚 用 Perplexity 帮我补充 这块" 紫色 按钮
- (c) 后端 SSE 帧 包含 `{"type":"kb_miss_hint","kb_id":"...","suggested_query":"...","reason":"..."}` (DevTools Network 可看)

**Fail**:
- 没出 紫色 卡片 → kb_miss_hint 没推 (检查 后端 chat 路径 / agent 是否 真没召回 chunks)
- 任何 AI 回复 都出 这卡片 → 条件 判错 (应该 仅 当 citations.length === 0)
