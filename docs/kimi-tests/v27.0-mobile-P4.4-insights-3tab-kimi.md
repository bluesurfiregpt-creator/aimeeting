## T-06 · 三 tab 切换 lazy load — 网络请求次数

### 前置 P-* PASS.

### 步骤
1. 打开 devtools Network panel, 清空.
2. 刷新 /m/insights.
3. 观察初次加载 + 切 tab 时的请求.

### Pass
- 初次加载: 只 1 个请求 `/api/m/insights?limit=50` (默认 Tab 1 lazy load 触发).
- 切到 Tab 2: 触发 1 个 `/api/m/tasks` 请求.
- 切到 Tab 3: 触发 1 个 `/api/memory?limit=100` 请求.
- 再切回 Tab 1: **不再发**新请求 (数据已缓存).
- 再切回 Tab 2: **不再发**新请求.

### Fail
- 初次加载发了 3 个请求 (eager load, 浪费流量).
- 每次切都重新拉.

### 必须复述
- Network 截图含所有 XHR 请求, 标 timestamp.
