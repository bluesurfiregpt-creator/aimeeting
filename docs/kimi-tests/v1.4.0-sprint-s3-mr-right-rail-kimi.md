## 3 · 用例 T-01 ~ T-05

### T-01 · MiraLive 真接 — 当前焦点 + 议程剩余 + 偏离风险

**前置**: P-2 + P-3 通

```bash
curl -s -b /tmp/cookies-leader.txt \
  https://aimeeting.zhzjpt.cn/api/meetings/<MEETING_ID>/agenda-progress | python3 -m json.tool
```

记下 `current_idx` + `items[current_idx].title` + `items[current_idx].time_budget_min` + `items[current_idx].elapsed_seconds`.

浏览器 用 leader cookie 进 `https://aimeeting.zhzjpt.cn/meeting/<MEETING_ID>/live`, 截图 右栏 "Mira 当下" section.

**Pass**:
- DOM 显 "当前焦点" 值 = `议程 ${current_idx+1} · ${items[current_idx].title}` (字面相等)
- DOM 显 "议程剩余" 值 = `${m} 分 ${ss} 秒` 格式 (跟 API 算的 budget - elapsed 一致, ±5 秒 容差)
- DOM 显 "偏离风险" 值 = `低 ✓` 或 `高 ⚠` (跟 timeline 最近 5 event 是否含 off_topic / stuck 一致)
- DOM 显 "正在说" 行 有 黄色 `演示` pill (data-testid="mr-mira-demo-mock"), 标 这是 mock 数据

**Fail**:
- "当前焦点" 显 "议程 2 · 搜索 A/B" 字符串 (旧 hardcoded) → 真接没生效 (RED)
- DOM 显 "演示数据" pill (data-testid="mr-mira-fallback-pill") → API 失败 (YELLOW, 需查后端 agenda-progress)
- "正在说" 行 没 演示 pill → 反幻觉违规 (RED)

**必须复述**:
- agenda-progress API 完整 JSON
- DOM 截图 + 4 row 字面 复述

### T-02 · DecisionPool 真接 — consensus 列表

**前置**: T-01 pass

```bash
curl -s -b /tmp/cookies-leader.txt \
  https://aimeeting.zhzjpt.cn/api/meetings/<MEETING_ID>/consensus | python3 -m json.tool
```

记下 array 长度 N + 每项 `agenda_idx / consensus_md / needs_human_review / dissents`.

DOM 截图 "决策池" section.

**Pass**:
- DOM section header 右侧 显 `${N} 条` (字面相等)
- N = 0 时 显 空态 "暂无决策 · AI 圆桌 收敛后 自动 沉淀" (data-testid="mr-decision-empty")
- N > 0 时:
  - DOM 卡片 数量 = N
  - 每卡 显 `已确认` 或 `待确认` (跟 `needs_human_review` 一致: false→已确认, true→待确认)
  - 每卡 显 `议程 ${agenda_idx + 1}`
  - `consensus_md` 截断 80 字 后 字面 出现在 DOM (前 80 字)
  - 含 `dissents.length > 0` 的 卡 显 "⚠ N 处 分歧"

**Fail**:
- DOM 显 `MR_DECISIONS` 旧 mock 3 条 (eg 标题 "采纳 vendor B" / "下周 1 前 sign-off") → 真接没生效 (RED)
- DOM count 跟 API N 不等 → 渲染 bug (RED)
- DOM 显 fallback pill (data-testid="mr-decision-fallback-pill") → API 失败 (YELLOW)

**必须复述**:
- consensus API 完整 JSON
- DOM 截图 + section header "N 条" 字面 + 第 1 卡 完整字段
