发版主题: 4 tab 全原生 + 微信 OAuth 一键登录 + 手机号一键登录 + 手机号注册 + 全局动效.

## 0. 死规矩 (反幻觉, Kimi 提交前 自检)

1. **只看真实数据**:每条 pass/fail 必须 基于 你 本步 实际看到的 UI / 抓到的 HTTP 响应,**不允许凭直觉补充**.
2. **复述必须原文**:有引号的字面值 (按钮文案 / Toast / 接口字段) **照抄**,标点不改,不缩写,不翻译.
3. **不允许跳步**:前一步 fail 时,后续依赖该步的用例全部标 `BLOCKED` 并解释,不允许"假设它过了".
4. **不允许编造账号**:只用本文档第 2 节列出的账号; 任何在 chat 里临时给的密码都不算数.
5. **每条必须给 4 段**:① 实际看到 (截图 + 关键文字原文) ② Pass / Fail / Blocked ③ Fail 时给具体理由 ④ 证据 (screenshot path / HTTP 响应原文).
6. **不允许出现 "应该 / 大概 / 估计 / 似乎"** 这类含糊词. 看到就是看到, 看不到就标 Fail / Blocked.

## 2. 预检 P-1 ~ P-5

> 任一 P-* fail → 后续 T-* 全部 BLOCKED.

### P-1 服务器健康
- 步骤: `curl -i https://aimeeting.zhzjpt.cn/healthz`
- Pass: HTTP 200 + body 含 `"ok":true`
- Fail: 5xx / timeout / 域名 解析 失败

### P-2 后端 鉴权 API 存活
- 步骤: `curl -X POST https://aimeeting.zhzjpt.cn/api/auth/token -H "Content-Type: application/json" -d '{"account":"x","password":"x"}'`
- Pass: HTTP 401 + body 含 `"detail":"incorrect account or password"`
- Fail: 5xx / 4xx 非 401 / 文案 不一致

### P-3 OAuth 配置
- 步骤: `curl -X POST https://aimeeting.zhzjpt.cn/api/auth/wx-login -H "Content-Type: application/json" -d '{"code":"fake_code"}'`
- Pass: HTTP 400 + body `detail` 含 `"errcode"` (说明后端能解析返回, 只是 fake code 被拒)
- Fail: HTTP 503 (说明 WX_APPID / WX_SECRET 没配) / 5xx / 无 errcode 字样

### P-4 phone 字段已 migration
- 步骤: 微信开发者工具登 `demo.lijg@futian.gov.cn` / `demo123` → 拿 token → `curl https://aimeeting.zhzjpt.cn/api/auth/me -H "Authorization: Bearer <token>"`
- Pass: HTTP 200 + body 有 `"email":"demo.lijg@futian.gov.cn"` 字段(phone 可为 null, 这是预期)
- Fail: HTTP 5xx / body 缺 email 字段 / 字段名错误

### P-5 微信开发者工具 启动
- 步骤: 打开微信开发者工具 → 项目目录指向 `wechat-miniprogram/` → 编译
