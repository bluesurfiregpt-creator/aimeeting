## 3 · 用例 T-01 ~ T-03

### T-01 · DB 字段 默认值 active

**前置**: P-1 pass

**目标**: 所有 老 数据 `status` 默认 'active', 不破坏.

```sql
SELECT status, COUNT(*) FROM meeting_agent_message GROUP BY status;
```

**Pass**: 至少 看到 一行 `status='active'`, 总数 ≥ 历史 message 数. 可选 一行 `'superseded'` 是 新 fix 触发 标的.
**Fail**: 看到 NULL 或 unknown value

**必须复述**: SQL 输出 完整 字符 (含 表头)

---

### T-02 · 至少 一条 message 触发 superseded

**前置**: P-3 pass (有 auto meeting), backend 部署 后 24h 内 跑过 至少 一个 auto meeting

```sql
SELECT id, agent_id, agenda_idx, status, superseded_by_message_id,
       LEFT(text, 80) AS text_preview
FROM meeting_agent_message
WHERE status = 'superseded'
ORDER BY created_at DESC LIMIT 5;
```

**Pass 条件**:
- 至少 1 行 `status = 'superseded'`
- `superseded_by_message_id` 非 NULL (指向 新 message.id)
- 新 message (`SELECT * FROM meeting_agent_message WHERE id = <superseded_by_message_id>`) 存在 + `agenda_idx` 跟 旧 一致

**Fail 条件**:
- 0 行 superseded (LLM judge 没触发 / 没冲突 模型 不识别 / fire-and-forget 失败)
- superseded_by_message_id NULL (update 没生效)

**注**: 若 没 conflict 触发, 说明 demo auto meeting 议程 太 平和 不出冲突, 改 跑 T-03 注入 冲突.

**必须复述**: SQL 输出 全部 列 (id / agent_id / agenda_idx / status / superseded_by / preview)

---

### T-03 · 主动 注入 冲突 — chat 模式 验证

**前置**: T-02 触发 0 (没自然冲突) 时 跑

由于 chat (1-on-1) 模式 不写 meeting_agent_message, 测 T-03 必须 走 会议室
@-mention 调用 2 个 立场对立 的 agent.

**步骤** (走 hybrid meeting):
1. 创建 hybrid meeting 用 leader
2. 输入框 @-mention `Stratos` 问 "我们 是否 应该 走 灰度 50% 流量?"
3. 等 Stratos 回 (5-15s, 假设 给 倾向 "推进" 答)
4. 输入框 @-mention `Lex` 问 "Stratos 刚才 建议 50%, 法务 上 有 红线 吗?"
5. 等 Lex 回 (5-15s, 法务 应 给 "暂缓" 倾向 — 立场对立)
6. 等 5-10s (fire-and-forget LLM judge ~2-5s)
7. 拉 transcript: `GET /api/m/meetings/{id}/transcript`
8. 检查 Stratos 那条 lines[i] 是否 `status = "superseded"`

**Pass 条件**:
- Stratos 第一条 `status: "superseded"`
- Stratos 那条 `superseded_by_message_id` = Lex 那条 的 `id`
- DOM 上 Stratos bubble 视觉 灰化 (opacity 0.5) + 标 "已被覆盖"

**Fail 条件**:
- 两条 都 `active` (LLM judge 没识别 立场对立)
- Stratos 那条 `superseded_by_message_id` 是 错 id

**注**: LLM judge 是 概率 — 若 试 1 次 没触发, 重复 2 次 看 命中率. 一致 0 触发 → 调高 prompt 严格度.

**必须复述**: API 返回 完整 JSON 含 status + superseded_by_message_id 两个字段
