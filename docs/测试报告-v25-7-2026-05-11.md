## ✅ Bug #4 / #5 / #6 验证细节(本次重点)

### Bug #4 — F-4 任务全闭环 现在跑通了

完整 E2E 复现(全部 200 / 状态正确):

```js
// ① leader 创建领导指令 → LLM 拆解
fetch("/api/me/directives", {method:"POST", credentials:"include",
  headers:{"Content-Type":"application/json"},
  body: JSON.stringify({content: "请房屋安全科本周完成 3 栋 D 级危房应急避险措施"})})
// → { id, drafts: [{ assignee_name: "房屋安全科", content: "完成3栋D级危房应急避险措施" }] }

// ② commit 派发给冯磊
fetch(`/api/me/directives/{id}/commit`, {method:"POST", credentials:"include",
  headers:{"Content-Type":"application/json"},
  body: JSON.stringify({tasks: [{
    content: "完成3栋D级危房应急避险措施",
    assignee_user_id: "1ecc73c6-…",  // 冯磊
    due_at: "..." }]})})
// → { committed_task_ids: ["4b571d22-…"], dispatched_count: 1 } ← 🎉 was 0

// ③ 用 reviewer 维度查得到任务
fetch("/api/me/tasks?role=reviewer&status=all").then(r=>r.json())
// → 任务 status="dispatched", assignee_user_id=冯磊, dispatched_at 有值

// ④ 切冯磊登录,/me 看到通知
//    『李建国 派发给你: 完成3栋D级危房应急避险措施』
//    任务出现在「待签收」tab,可点 签收 / 退回

// ⑤ 冯磊 签收 → 开始 → 提交
fetch(`/api/me/tasks/{id}/accept`, {method:"POST"})  // → status=accepted
fetch(`/api/me/tasks/{id}/start`, {method:"POST"})   // → status=in_progress
fetch(`/api/me/tasks/{id}/submit`, {method:"POST",
  body: JSON.stringify({completed:"…",issues:"…",next:"…",evidence:"…"})})
// → status=submitted

// ⑥ 切回 leader,/me 看到 「冯磊 上报办结申请」 通知,点 approve
fetch(`/api/me/tasks/{id}/approve`, {method:"POST"})
// → status=done

// ⑦ 30 秒后查 KB → 自动沉淀的文档出现 🎉
// 综合事务AI专家 KB 多了一篇:
// "[自动沉淀 20260510-1711] 完成3栋D级危房应急避险措施.md"  status=ready
```

整条 v24 / v25 任务闭环 demo **可以演示了**。
